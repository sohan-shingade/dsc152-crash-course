Question 1
P-Value Interpretation • Wording Trap

A researcher tests $H_0: \mu = 20$ vs $H_A: \mu \neq 20$ and gets $p = 0.14$.

Select ALL correct interpretations of this p-value:

Correct: A, D, E

(A) TRUE — This is the definition of a p-value.
(B) FALSE — P-value is NOT $P(H_0 \text{ is true})$. Classic trap.
(C) FALSE — Type I error rate = $\alpha$, not the p-value.
(D) TRUE — $0.14 > 0.05$, so fail to reject.
(E) TRUE — $0.14 > 0.10$, so also fail to reject at $\alpha = 0.10$.
(F) FALSE — P-value is not $P(H_A \text{ is true})$ either.
Exam wording: "The p-value is the probability of observing a test statistic as extreme or more extreme than the one observed, assuming $H_0$ is true."
#1 exam trap: The p-value is NOT the probability that $H_0$ is true, NOT the probability that $H_A$ is true, NOT the probability of Type I error.
Question 2
t-Test Conditions • Type I Error

You run t.test(data, mu = 6) on $n = 5$ observations from a highly right-skewed distribution at $\alpha = 0.05$.

(a) State the two conditions for the t-test to be valid.
(b) What happens to the actual Type I Error rate when these conditions are violated with skewed data and small $n$?
(a) Two conditions:
1. Random sample — observations are independent and representative of the population.
2. Data come from a normal distribution (or $n$ is large enough for CLT) — so the test statistic follows a $t$-distribution under $H_0$.
(b) Correct: Inflated above 0.05.

From lecture simulations: with Gamma(skewed) data and $n=5$, the actual Type I Error was ~0.10-0.11 — more than TWICE the nominal 0.05. The t-test rejects too often because the $t$-distribution doesn't match the actual sampling distribution.

Question 3
Write R Code • Type I Error Simulation

Write the complete R code to estimate the Type I Error rate of a t-test when data come from a $\text{Gamma}(\text{shape}=1.2, \text{scale}=6/1.2)$ distribution (which has true mean $= 6$). Use 10,000 simulations, $n = 5$, $\alpha = 0.05$.

type1_sim.R
count <- 0
for (i in 1:10000) {
  gam_data <- rgamma(n = 5, shape = 1.2, scale = (6 / 1.2))
  p.val <- t.test(gam_data, mu = 6)$p.value
  if (p.val < 0.05) {
    count <- count + 1
  }
}
TypeI <- count / 10000  # ~0.10-0.11 (inflated!)
Key pattern: Generate data under $H_0$ being TRUE (mean = 6), run the test, count rejections. The proportion of rejections = estimated Type I Error rate. This pattern appears on every quiz.
Question 4
Sign Test • Computation

Data: $\{5.5, 4.8, 6.1, 3.2, 4.9, 1.7, 7.2\}$. Test $H_0$: median $= 4$ vs $H_A$: median $\neq 4$.

(a) The sign test tests the _______ (mean / median). Why does this matter?
(b) Count the number of observations above and below 4. Write the R code for the p-value.
(c) The achievable significance levels for $n = 7$ jump from 0.016 to 0.125. Explain why you can't get exactly $\alpha = 0.05$.
(a) The sign test tests the median, not the mean. This matters because for skewed distributions, mean $\neq$ median, so the t-test and sign test are testing different things and can give different results.

(b) Signs relative to 4: $+, +, +, -, +, -, +$ → 5 above, 2 below.
sign_test.R
n_pos <- 5; n_neg <- 2; n_eff <- 7
p_value <- 2 * pbinom(min(n_pos, n_neg), size = n_eff, prob = 0.5)
p_value <- min(1, p_value)
# = 2 * pbinom(2, size=7, prob=0.5) = 2 * 0.227 = 0.453
(c) The Binomial distribution is discrete. With $n = 7$ and $p = 0.5$, the achievable tail probabilities are fixed values: $2 \times P(X \leq k)$ for $k = 0, 1, 2, \ldots$ These jump from 0.016 ($k=0$) to 0.125 ($k=1$), skipping 0.05 entirely.
Key: pbinom(q, size, prob) = $P(X \leq q)$. For $P(X \geq k)$ use pbinom(k-1, size, prob, lower.tail=FALSE). Getting the off-by-one right matters.
Question 5
Power • Four Factors

Select ALL changes that INCREASE statistical power:

Correct: A, B, E

(A) TRUE — More data = more power.
(B) TRUE — Bigger real effect = easier to detect.
(C) FALSE — More noise = harder to detect signal. Power goes DOWN.
(D) FALSE — Stricter threshold = harder to reject = less power.
(E) TRUE — One-sided puts all rejection probability on one side, making it easier to reject in that direction.
Exam-ready: Power increases with $n \uparrow$, $\delta \uparrow$, $\sigma \downarrow$, $\alpha \uparrow$.
Question 6
Write R Code • Power Simulation + Theory

You want to estimate the power of a one-sample t-test. True mean $= 22$, null mean $= 20$ (so $\delta = 2$), $\sigma = 5$, $n = 30$.

(a) Write the power.t.test() call.
(b) Now suppose the data come from a Gamma distribution (not normal). Can you still use power.t.test()? Why or why not?
(c) Write a simulation-based power function that works for ANY distribution.
(a)
power_theory.R
power.t.test(n = 30, delta = 2, sd = 5, sig.level = 0.05,
             type = "one.sample", alternative = "two.sided")
(b) No. power.t.test() assumes data are normally distributed. With Gamma data, there's no closed-form solution for the distribution of the t-statistic. Must use simulation.

(c)
power_sim.R
sim_power <- function(n, delta, sd, reps = 10000) {
  count <- 0
  for (i in 1:reps) {
    samp <- rnorm(n = n, mean = delta, sd = sd)  # or rgamma, etc.
    pval <- t.test(samp, mu = 0)$p.value
    if (pval < 0.05) {
      count <- count + 1
    }
  }
  return(count / reps)
}
Key difference from Type I Error sim: Type I Error: generate under $H_0$ true (mean = null value). Power: generate under $H_A$ true (mean = null value + $\delta$). Same loop structure, different mean argument.
Question 7
Significance vs Effect Size • Applied Scenario

A study with $n = 50{,}000$ tests whether average TSA wait time differs from 20 min. Results: $\bar{x} = 20.3$, $s = 15$, $p < 0.0001$. The 95% CI is $[20.17, 20.43]$.

(a) Should you reject $H_0$ at $\alpha = 0.05$?
(b) Is this result practically meaningful? Why or why not?
(c) What concept from Lecture 5 does this illustrate?
(a) Yes, reject $H_0$. $p < 0.05$ means we reject. The statistical decision is based on the p-value, period.

(b) NOT practically meaningful. The actual difference is only 0.3 minutes (18 seconds). No traveler would notice or care. The CI $[20.17, 20.43]$ shows the true mean is very close to 20.

(c) The distinction between statistical significance and practical (effect-size) significance.
Exam wording: "With very large $n$, even tiny, meaningless differences become statistically significant. You must also look at the estimated effect size and ask if it is big enough to care about."
Related trap: "Failing to reject $H_0$ does NOT mean $H_0$ is true." It might just mean you don't have enough power (small $n$, large $\sigma$). Absence of evidence $\neq$ evidence of absence.
Question 8
Bootstrap CI • Type I Error

A colleague constructs a 95% bootstrap CI from a sample and says: "Bootstrap is non-parametric, so its Type I Error rate is guaranteed to be exactly 0.05."

(a) Is this claim correct?
(b) How would you check the bootstrap CI's Type I Error rate? Describe the simulation.
(a) No. Bootstrap CIs can have inflated or deflated Type I Error, especially for small samples. Being non-parametric does NOT guarantee correct Type I Error.

(b) Simulation design:
bootstrap_type1.R
# Generate data under H0 (true mean = 20)
# For each sample, build bootstrap CI
# Check if 20 falls OUTSIDE the CI
# Proportion outside = estimated Type I Error

count <- 0
for (i in 1:10000) {
  samp <- rnorm(n = 10, mean = 20, sd = 5)  # H0 is true
  boot_means <- replicate(5000, mean(sample(samp, replace = TRUE)))
  ci <- quantile(boot_means, probs = c(0.025, 0.975))
  if (20 < ci[1] || 20 > ci[2]) {
    count <- count + 1  # rejected H0 (incorrectly)
  }
}
type1_boot <- count / 10000

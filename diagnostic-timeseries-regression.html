Question 1
Conceptual • LINE Conditions

Recall the LINE conditions for valid linear regression inference:

  • Linearity
  • Independence of observations
  • Normality of residuals
  • Equal variance (homoscedasticity)

When you use standard lm() on time series data (e.g., a player's points scored across 82 games in order), which LINE condition is most likely violated, and why?

Correct: I — Independence
In time series data, consecutive observations are typically correlated — a player who scored 40 last game may be more likely to score high again (momentum, fatigue, matchup streaks). This autocorrelation violates the Independence condition.

This is the central motivation for Lectures 18-19. Standard lm() assumes every row is independent, but time series rows are ordered and correlated. Ignoring this structure produces wrong standard errors and p-values.

Why it matters: When independence is violated, the standard errors from lm() are typically too small, making p-values too small, inflating Type I error OR (as shown in class) reducing power when the autocorrelation masks real effects.
Question 2
Model Structure • AR vs MA Errors

In time series regression, we model:

yt = β0 + β1·xt + ηt

Instead of assuming ηt are independent (like regular regression), we allow the error term to have time structure.

(a) If ηt = εt + θ1·εt-1, what type of model does the error follow? Name it and state its order.
(b) What is the key difference between AR and MA error structures? When a "shock" (unexpected event) hits the series, how long does its effect last in each?
(a) Correct: MA(1) — Moving Average order 1
The error depends on the current εt and the previous error εt-1. This is the MA(1) structure. Compare: AR(1) would be ηt = φ1·ηt-1 + εt (error depends on its own previous value, not the previous innovation).
(b) Shock persistence:
AR: Shocks linger for many time steps. In AR(1), y_t depends on y_{t-1}, which depended on y_{t-2}, etc. A shock propagates forward indefinitely (though decaying). MA: Shocks last exactly q time steps. In MA(1), y_t depends on ε_t and ε_{t-1} only. After 1 step, the shock is gone from the model.
Memory trick: AR = long memory (shock echoes), MA = short memory (shock dies after q steps).
Question 3
R Code • Arima() Syntax

A researcher studies whether opponent defensive rating (opp_def) predicts points scored (pts) across an NBA season (82 games in order). They suspect the errors have an MA(1) structure.

(a) Write the R code to fit this time series regression using Arima(). Use the correct order argument.
(b) In the order = c(p, d, q) argument, what does each of the three numbers represent?
(c) How do you get p-values from the fitted Arima model? (Hint: summary() does NOT give you p-values for Arima objects.)
(a) R code:
library(forecast) model <- Arima(y = nba_df$pts, order = c(0, 0, 1), xreg = nba_df$opp_def)
Key details: order = c(0, 0, 1) means AR(0), differencing(0), MA(1). The predictor goes in xreg, not in a formula like lm().
(b) The three numbers:
p = AR order (autoregressive terms) d = differencing order (not used in this course, typically 0) q = MA order (moving average terms) So order = c(0, 0, 1) means: no AR, no differencing, MA(1) errors order = c(2, 0, 0) means: AR(2) errors, no differencing, no MA
(c) Correct: coeftest(model) from lmtest
library(lmtest) coeftest(model)
Exam trap: Unlike lm(), the summary() of an Arima object does NOT show p-values. You must use coeftest(). Easy to forget on the exam.

(a) Both models find opp_def significant, but the p-values differ (0.010 vs 0.001). Which model's p-value should you trust more, and why?
(b) What does the ma1 = -0.55 row in Model B represent?
(c) Notice Model B uses z values while Model A uses t values. This is the same difference you saw in another type of regression this quarter. Which one?
(a) Trust Model B (Arima).
Model B properly accounts for the MA(1) autocorrelation in the errors. Model A (lm) assumes independence, which is violated here. By correctly modeling the error structure, Model B produces more accurate standard errors and therefore a more trustworthy p-value.

In this case, Model B actually finds a more significant result. This illustrates the key lesson from Lec 19: ignoring time dependency can cause loss of power — you may miss real effects.

(b) The ma1 coefficient:
ma1 = -0.55 is the θ1 parameter in the MA(1) error structure: ηt = εt + (-0.55)·εt-1. It means about 55% of the previous game's random deviation (with reversed sign) carries forward into the current game's error.
(c) Correct: Logistic regression
Connection: Both logistic regression (glm) and Arima use z-values instead of t-values in their output. The lm() function is the only one that uses t-values. This is a pattern worth remembering for reading output tables on the exam.
Question 5
Power • Simulation Design • Key Takeaway

Lecture 19 runs a power simulation comparing lm() vs Arima() on time series data. The true model has AR(2) errors:

yt = β0 + β1·xt + φ1·yt-1 + φ2·yt-2 + εt

with β1 = 2 (a real effect exists), φ1 = 0.65, φ2 = 0.25.

(a) In 1000 simulations, which method will have higher power (reject H0 more often when H0 is false)?
(b) Write the R code to fit the correct Arima model for this scenario, where y is the outcome and time is the predictor.
(c) Complete this sentence with the key takeaway from Lecture 19:

"Failing to account for time dependency in your data can lead to ___________."

(a) Correct: Arima() has higher power.
When the data truly has AR(2) structure, Arima(order=c(2,0,0), xreg=...) correctly accounts for it and can detect β1 = 2 more often. lm() ignores the autocorrelation, produces incorrect standard errors, and frequently MISSES the real effect.
(b) R code:
library(forecast) library(lmtest) model <- Arima(y, order = c(2, 0, 0), xreg = time) coeftest(model)

order = c(2, 0, 0) = AR(2) errors, no differencing, no MA terms.

(c) Correct: "incorrect inference (and in particular, loss of power)"
This is the direct quote from the lecture: "Failing to account for time dependency in your data can lead to incorrect inference. In every example here, we showed a loss of power, but other things could happen."
Nuance: The lecture emphasizes loss of power (missing real effects), but acknowledges other problems can occur too. The coefficient estimates themselves may still be roughly unbiased — it's the standard errors and p-values that go wrong.

Part A Learn: Hypothesis Testing & the t-Test
01 What is hypothesis testing? (The courtroom analogy)

Hypothesis testing works like a criminal trial:

Criminal Trial

Defendant is presumed innocent

Prosecution presents evidence

Jury asks: "Is the evidence strong enough to convict?"

Verdict: guilty or not guilty (never "innocent")

Hypothesis Test

$H_0$ is presumed true

Data is the evidence

We ask: "Is the data surprising enough to reject $H_0$?"

Decision: reject $H_0$ or fail to reject $H_0$ (never "accept $H_0$")

Never say "accept $H_0$." Just like a jury says "not guilty" (not "innocent"), we say "fail to reject" — we didn't find enough evidence, but that doesn't mean $H_0$ is true.

02 $H_0$, $H_A$, and the p-value — what they actually mean
$H_0$ (null hypothesis): The boring claim. "Nothing interesting is happening." Always has an $=$ sign.
$H_A$ (alternative): The interesting claim. "Something IS going on." Has $\neq$, $>$, or $<$.

Example: UCSD claims students sleep 6 hours on average.

$$H_0: \mu = 6 \qquad H_A: \mu \neq 6$$

Now, what IS a p-value? Let's build it from scratch:

You collect data: sleep hours = {3, 7, 1, 2, 2}. Sample mean $\bar{x} = 3.0$. That's pretty far from 6. But is it far enough?

The p-value answers: "If the true mean really were 6, how often would we see data this extreme just by random chance?"

  • Small p-value (e.g., 0.01) → "This data would be really rare if $H_0$ were true" → reject $H_0$
  • Large p-value (e.g., 0.40) → "This data isn't unusual under $H_0$" → fail to reject
The sentence to memorize:
"The p-value is the probability of observing a test statistic as extreme or more extreme than the one observed, assuming $H_0$ is true."

That last part — "assuming $H_0$ is true" — is what makes all the wrong interpretations wrong. The p-value is conditional on $H_0$. It does NOT tell you the probability that $H_0$ is true or false.

Three things the p-value is NOT:
✗ The probability that $H_0$ is true
✗ The probability that $H_A$ is true
✗ The probability of making a Type I error
03 The t-test: formula, R code, and when it works

The one-sample t-test checks whether a population mean equals some value $\mu_0$.

The test statistic

$$t_s = \frac{\bar{x} - \mu_0}{s / \sqrt{n}}$$

In words: how many standard errors is our sample mean from the hypothesized mean?

  • $\bar{x}$ = sample mean
  • $\mu_0$ = the null hypothesis value
  • $s$ = sample standard deviation
  • $n$ = sample size

Big $|t_s|$ → sample mean is far from $\mu_0$ → small p-value → reject.

R code — three ways

t_test.R
# Method 1: Built-in function (use this on exams)
t.test(sleep, mu = 6)

# Extract just the p-value
t.test(sleep, mu = 6)$p.value

# Method 2: Manual calculation
t_s <- (mean(sleep) - 6) / (sd(sleep) / sqrt(length(sleep)))

# Method 3: One-sided test
t.test(sleep, mu = 6, alternative = "less")     # HA: mu < 6
t.test(sleep, mu = 6, alternative = "greater")  # HA: mu > 6
Memorize: t.test(data, mu = null_value)$p.value
The $p.value accessor is on every quiz. Not $p, not $pvalue. Exactly $p.value.

Two conditions for the t-test to be valid

  1. Random sample — observations are independent and representative of the population
  2. Data come from a normal distribution (or $n$ is large enough for CLT) — so the test statistic follows a $t$-distribution under $H_0$

If these are violated (especially with skewed data and small $n$), the t-test can break. That's Module 2.

04 Making the decision: reject or fail to reject

The rule is simple:

$$\text{p-value} < \alpha \implies \text{reject } H_0 \qquad \text{p-value} \geq \alpha \implies \text{fail to reject } H_0$$

$\alpha$ is the significance level — usually 0.05, but read the question. Quiz 1 used $\alpha = 0.10$ for one question. If you use 0.05 when they said 0.10, you lose points.

One-sided from two-sided

If you have a two-sided p-value and want one-sided:

  • If $\bar{x}$ is in the direction of $H_A$: one-sided $p = \text{two-sided } p / 2$
  • If $\bar{x}$ is in the opposite direction: one-sided $p = 1 - \text{two-sided } p / 2$

Example from Quiz 1: Two-sided $p = 0.14$, $\bar{x} < \mu_0$, $H_A: \mu < \mu_0$. Same direction → one-sided $p = 0.14/2 = 0.07$. At $\alpha = 0.10$: $0.07 < 0.10$ → reject.

Confidence interval method: If the 95% CI contains $\mu_0$ → fail to reject at $\alpha = 0.05$. If it doesn't contain $\mu_0$ → reject. Same conclusion as the p-value, always.

05 Exam Question Type: "Interpret the p-value"
📋 Real question (Quiz 1, Q1c):
p-value = 0.14. A colleague says: "There is a 14% probability the true average is $4.00." Is this correct?

✗ Wrong answer

"Yes, it means there's a 14% chance the mean is $4."

Confuses "probability of data given H0" with "probability of H0 given data." These are completely different things.

✓ Full-marks answer

"No. The p-value is the probability of observing a test statistic as extreme or more extreme than the one observed, assuming $H_0$ is true. It is NOT the probability that the true mean is $4."

Wording keys the grader looks for:
✓ "assuming $H_0$ is true" — the conditional
✓ "as extreme or more extreme" — tail probability
✓ "test statistic" — not "the mean" or "the data"
✓ Explicitly say what it's NOT if the question gives a wrong interpretation

$p = 0.03$ for $H_0: \mu = 50$. Write the correct interpretation in one sentence.

"If $H_0$ is true (i.e., if the true mean is 50), there is a 3% probability of observing a test statistic as extreme or more extreme than the one observed."

Check your answer for: Did you say "assuming/if $H_0$ is true"? Did you say "as extreme or more extreme"? If yes, full marks.
06 Exam Question Type: "State the conditions for the t-test"
📋 Real question (Quiz 1, Q1e):
State the two conditions for the t-test to produce valid inference.

✗ Loses points

"Data should be normal and sample should be big enough."

Misses "random/independent sample" entirely. And "big enough" is vague — the condition is normality OR large n for CLT.

✓ Full marks

1. Random sample — observations are independent and representative of the population.
2. Data from a normal distribution (or $n$ large enough for CLT) — so the test statistic follows a $t$-distribution under $H_0$.

Don't confuse the condition sets:
t-test: 2 conditions (random sample + normality)
Linear regression: 4 conditions (LINE)
Logistic regression: 3 conditions (binary + independence + linearity on log-odds)
These are DIFFERENT. Module 10 covers LINE, Module 15 covers logistic conditions.
✏️ Your Turn

You run a t-test on $n = 5$ observations from a right-skewed distribution. Which condition is violated? What happens to the Type I Error rate?

The normality condition is violated (right-skewed, $n$ too small for CLT). The actual Type I Error rate is inflated above $\alpha$ — the test rejects more often than it should. With $n = 5$ and Gamma data, lecture simulations showed ~10-11% when it should be 5%.
07 Exam Question Type: "Write the R code for..."
📋 Real question (Practice Quiz 1, Q1b):
TSA data. Airport claims average wait = 20 min. Write R code to get the p-value.

✗ Loses points

t.test(tsa$wait_min, 20)

Missing mu =. And doesn't extract the p-value.

✓ Full marks

t.test(tsa$wait_min, mu = 20)$p.value

R syntax cheat sheet for hypothesis testing:
t.test(x, mu = val) — run the test
t.test(x, mu = val)$p.value — extract p-value
t.test(x, mu = val)$conf.int — extract CI
t.test(x, mu = val, alternative = "greater") — one-sided

Must name the mu argument. t.test(x, 6) does NOT work — it thinks 6 is a second dataset.
✏️ Your Turn

Data: delivery$tip_amount. Test whether average tip differs from $4. Get the p-value. Then test whether it's less than $4 (one-sided).

answer.R
# Two-sided
t.test(delivery$tip_amount, mu = 4)$p.value

# One-sided (less than)
t.test(delivery$tip_amount, mu = 4, alternative = "less")$p.value
Part C Quick Quiz — Can You Apply This?
Q1 Quiz: 4 questions, exam format
Question 1 of 4

A researcher gets $p = 0.08$ for $H_0: \mu = 100$. Colleague says: "There's a 92% chance the true mean is not 100." Is this correct? If not, give the right interpretation.

No. The p-value is not $1 - P(H_0)$. Correct: "If $H_0$ is true, there is an 8% probability of observing a test statistic as extreme or more extreme than the one observed."
Question 2 of 4

Two-sided $p = 0.08$. The sample mean is above $\mu_0$. You want $H_A: \mu > \mu_0$.
(a) One-sided p-value?
(b) Decision at $\alpha = 0.05$?
(c) Decision at $\alpha = 0.10$?

(a) $0.08 / 2 = 0.04$ (same direction as $H_A$, so divide by 2)
(b) $0.04 < 0.05$ → reject $H_0$
(c) $0.04 < 0.10$ → reject $H_0$
Question 3 of 4

Write the R code: test whether the mean of scores differs from 75. Extract the p-value. Also get the 95% CI.

t.test(scores, mu = 75)$p.value
t.test(scores, mu = 75)$conf.int
Question 4 of 4

95% CI for the mean is $[72.3, 78.1]$. Does this interval suggest we reject $H_0: \mu = 75$ at $\alpha = 0.05$? What about $H_0: \mu = 70$?

$\mu = 75$: 75 is INSIDE [72.3, 78.1] → fail to reject.
$\mu = 70$: 70 is OUTSIDE [72.3, 78.1] → reject $H_0$.

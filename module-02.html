Part A Learn: Type I Error & When Tests Break
01 What IS a Type I Error? (The fire alarm analogy)

There are two ways to be wrong in hypothesis testing:

Type I Error (False alarm)

You reject $H_0$ when $H_0$ is actually true.

Like a fire alarm going off when there's no fire.

Rate should equal $\alpha$ (usually 0.05).

Type II Error (Missed fire)

You fail to reject $H_0$ when $H_0$ is actually false.

Like a fire alarm NOT going off when there IS a fire.

Related to power: Power $= 1 - P(\text{Type II})$.

This module focuses on Type I Error — specifically, what happens when a test's Type I Error rate isn't actually $\alpha$.

Validity: A test is "valid" when its actual Type I Error rate equals $\alpha$. An invalid test is like a broken fire alarm — it goes off too much (inflated) or not enough (deflated).
02 When the t-test breaks: skewed data + small $n$

The t-test assumes the sampling distribution of $\bar{x}$ is bell-shaped. This is true when:

  • The data itself is normally distributed (any $n$), OR
  • $n$ is large enough for the Central Limit Theorem to kick in (roughly $n \geq 30$)

What if neither is true? Say you have $n = 5$ from a right-skewed distribution like the Gamma:

Left: Normal distribution (symmetric). Right: Gamma distribution (right-skewed). With $n = 5$, the t-test assumes the left shape but gets the right shape.

Think of it like a metal detector calibrated for a flat beach. Take it to a rocky beach and it beeps more often — not because there's more metal, but because the calibration is wrong for this terrain.

The t-test's "calibration" (the $t$-distribution) assumes symmetry. Skewed data makes extreme $t_s$ values more common than the $t$-distribution expects. So the test says "significant!" more often than it should.

Result: With skewed data and small $n$, the Type I Error rate is inflated above $\alpha$. Lecture simulations: Gamma data, $n = 5$, $\alpha = 0.05$ → actual Type I Error ≈ 0.10–0.11. That's double!

Why does large $n$ fix it? The CLT says: no matter what the data distribution looks like, $\bar{x}$ becomes approximately normal for large $n$. The rocky beach gets smoothed out.

Memorize the direction: skewed data + small $n$ → Type I Error INFLATED (too many false alarms, rejects too often). Not deflated. This tripped you up on the diagnostic.
03 The Type I Error simulation — the pattern you'll use forever

How do you measure a test's Type I Error rate? You can't do it with one dataset. You need a simulation:

  1. Generate data where $H_0$ is TRUE (you know the right answer)
  2. Run the test, get the p-value
  3. Check: did the test reject? (p-value $< \alpha$?)
  4. Repeat 10,000 times
  5. Proportion of rejections = estimated Type I Error rate

If the test is valid, this proportion should be ≈ $\alpha$. If it's inflated (like 0.10 when $\alpha = 0.05$), the test is broken.

Here's the code, annotated line by line:

type1_error_sim.R
# GOAL: Check if t-test has correct Type I Error rate
# for Gamma(shape=1.2, scale=5) data with n=5

count <- 0                        # counter for rejections

for (i in 1:10000) {              # repeat 10,000 times

  # Step 1: Generate data under H0 TRUE
  # Gamma with shape=1.2, scale=5 has mean = 1.2 * 5 = 6
  # We test H0: mu = 6, so H0 IS true here
  samp <- rgamma(n = 5, shape = 1.2, scale = 5)

  # Step 2: Run the test, extract p-value
  p.val <- t.test(samp, mu = 6)$p.value

  # Step 3: Did it (incorrectly) reject?
  if (p.val < 0.05) {
    count <- count + 1            # false alarm!
  }
}

# Step 4: Proportion of false alarms
TypeI <- count / 10000            # should be ~0.05 if test is valid
                                  # actual result: ~0.10 (INFLATED!)
The key line is the mean. In a Type I Error simulation, you generate data where $H_0$ IS true. So the mean of your generated data EQUALS the null value. If the question says $H_0: \mu = 6$, you generate data with mean $= 6$.

Contrast with power simulation (Module 4): there you generate data with mean $\neq$ null value because $H_A$ is true.

04 Parametric vs non-parametric: what these words actually mean

Parametric (e.g., t-test)

Relies on distributional assumptions (normality) for the math to work.

Pro: More powerful when conditions met — uses the shape of the distribution to squeeze out more information.

Con: Can break (inflated Type I Error) when assumptions are violated.

Non-parametric (e.g., sign test, permutation, bootstrap)

Does NOT assume a specific distribution.

Pro: Works for any data distribution — no normality needed.

Con: Less powerful (harder to detect real effects). Sign test also has the discreteness problem (Module 3).

Common trap: "Non-parametric = always better" — WRONG. Non-parametric tests have less power when the parametric conditions actually hold. HW1 showed: t-test power ≈ 0.75 vs sign test power ≈ 0.55 on normal data. You're trading power for robustness.
Another trap: "Bootstrap is non-parametric, so it guarantees correct Type I Error." — WRONG. Bootstrap CIs can have inflated or deflated Type I Error, especially for small samples. Non-parametric $\neq$ guaranteed valid.
05 Bootstrap CIs: how they work and why they're not bulletproof

A bootstrap CI is a non-parametric way to build a confidence interval:

  1. Resample your data WITH replacement (same size as original)
  2. Compute the mean of the resample
  3. Repeat 10,000 times → you get 10,000 "bootstrap means"
  4. Take the 2.5th and 97.5th percentiles → 95% CI
bootstrap_ci.R
# Build bootstrap 95% CI
boot_means <- replicate(10000, mean(sample(data, replace = TRUE)))
ci <- quantile(boot_means, probs = c(0.025, 0.975))

# Decision: if null value is OUTSIDE the CI → reject H0
# If null value is INSIDE the CI → fail to reject

How to use it as a hypothesis test: If $\mu_0$ is outside the 95% CI → reject $H_0$ at $\alpha = 0.05$. If inside → fail to reject.

How to check bootstrap Type I Error (asked on Quiz 1)

Same simulation idea as the t-test, but now you build a CI instead of computing a p-value:

bootstrap_type1.R
count <- 0
for (i in 1:10000) {
  # Generate under H0 TRUE (mean = 20)
  samp <- rnorm(n = 10, mean = 20, sd = 5)

  # Build bootstrap CI
  boot_means <- replicate(5000, mean(sample(samp, replace = TRUE)))
  ci <- quantile(boot_means, probs = c(0.025, 0.975))

  # Check: does CI (incorrectly) exclude 20?
  if (20 < ci[1] || 20 > ci[2]) {
    count <- count + 1    # false rejection
  }
}
type1_boot <- count / 10000
From Quiz 1 answer key: "The bootstrap can have an inflated or deflated Type I error rate. The accuracy of bootstrap confidence intervals relies on the sample being an accurate representation of the population, which is not always the case."

✗ Wrong

$\alpha$ is what you WANT. Actual Type I Error is what you GET. They're not the same when conditions are violated. And skewed + small $n$ → inflated, not deflated.

The sentence: "The Type I Error rate is likely to be inflated above $\alpha$ because the normality condition is violated and $n$ is too small for CLT."

08 Exam Question Type: "Is bootstrap guaranteed correct?"
📋 Real question (Quiz 1, Q3c):

✗ Trap answer

"Correct — bootstrap makes no distributional assumptions, so it works perfectly."

✓ From Quiz 1 answer key

"The bootstrap can have an inflated or deflated Type I error rate."

"The accuracy of bootstrap confidence intervals relies on the sample being an accurate representation of the population, which is not always the case."

Why bootstrap can fail: Bootstrap resamples from your sample. If your sample is small or doesn't represent the population well (e.g., you missed the tail of a skewed distribution), the bootstrap CI inherits that problem. Non-parametric ≠ assumption-free. The assumption is that your sample looks like the population.

Q Quiz: 4 questions
Q1

Define Type I Error in one sentence. What should its rate equal?

Rejecting $H_0$ when $H_0$ is true (false positive). Its rate should equal $\alpha$ (the significance level).
Q2

You simulate: generate from $\text{Gamma}(\text{shape}=3, \text{scale}=2)$ [mean $= 6$], run t.test(samp, mu = 6), count rejections at $\alpha = 0.05$ over 10,000 reps. You get 0.08. What does 0.08 represent and is this test valid?

0.08 is the estimated Type I Error rate. It is inflated above 0.05, so the test is NOT valid for this data distribution at $n$ = whatever was used. The t-test rejects $H_0$ 8% of the time when it should only reject 5%.
Q3

What's the difference between a parametric and non-parametric test? Give one pro and one con of each.

Parametric (e.g., t-test): relies on distributional assumptions. Pro: more powerful when conditions met. Con: can break (inflated Type I Error) when violated.
Non-parametric (e.g., sign test, bootstrap): no distributional assumptions. Pro: works for any distribution. Con: less powerful; bootstrap can still have inflated/deflated Type I Error for small samples.

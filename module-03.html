Part A Learn: The Sign Test
01 Why do we need a sign test when we have the t-test?

Module 2 showed: the t-test breaks with skewed data and small $n$. So what do you do when your data IS skewed and $n$ IS small?

Option 1: Use a bigger sample (not always possible).

Option 2: Use a test that doesn't assume normality.

That's the sign test. It's non-parametric — it works on ANY distribution. But there's a catch:

The sign test tests the MEDIAN, not the mean. The t-test tests the mean. For symmetric distributions (like Normal), mean = median, so it doesn't matter. For skewed distributions, mean $\neq$ median — and the two tests can give opposite answers.

From HW1: minutes played data ($n = 32$):

  • t-test: $p = 0.035$ → reject $H_0$
  • Sign test: $p = 0.377$ → fail to reject $H_0$

Different results because they're testing different things. The t-test found the mean differs from the null; the sign test didn't find evidence that the median differs.

02 How the sign test works — step by step with real numbers

Let's walk through it with real data from Quiz 1, Q2:

Data: $\{5.5,\; 4.8,\; 6.1,\; 3.2,\; 4.9,\; 1.7,\; 7.2\}$. Test $H_0$: median $= 4$ vs $H_A$: median $\neq 4$.

Step 1: Subtract the null value from every observation

5.5 - 4 = +1.5 → + 4.8 - 4 = +0.8 → + 6.1 - 4 = +2.1 → + 3.2 - 4 = -0.8 → − 4.9 - 4 = +0.9 → + 1.7 - 4 = -2.3 → − 7.2 - 4 = +3.2 → +

Step 2: Count positives and negatives

$n_+ = 5$ (above null) $n_- = 2$ (below null) $n_{\text{eff}} = 5 + 2 = 7$ (ignoring any ties with exactly 4)

Step 3: Under $H_0$, each observation is equally likely to be above or below

If the true median is 4, then any observation has a 50/50 chance of being above or below 4. So:

$$n_+ \sim \text{Binomial}(n_{\text{eff}},\; 0.5) \quad \text{under } H_0$$

Step 4: Compute two-sided p-value

$$p\text{-value} = 2 \times P(X \leq \min(n_+, n_-)) = 2 \times P(X \leq 2)$$
sign_test_pvalue.R
p_value <- 2 * pbinom(min(5, 2), size = 7, prob = 0.5)
p_value <- min(1, p_value)   # cap at 1

# = 2 * pbinom(2, size = 7, prob = 0.5)
# = 2 * 0.2266
# = 0.4531  →  fail to reject (0.45 > 0.05)
The sign test formula (memorize):
p_value <- min(1, 2 * pbinom(min(n_pos, n_neg), size = n_eff, prob = 0.5))

Use $\min(n_+, n_-)$ for two-sided test. The min(1, ...) caps it at 1 since $2 \times P$ can exceed 1 when $n_+$ and $n_-$ are close.
03 The pbinom off-by-one trap (this WILL be on the exam)

R's pbinom is the source of most sign test errors. Here's what each call gives you:

You wantMathR codeWatch out
$P(X \leq k)$ CDF, includes $k$ pbinom(k, size, 0.5) Default — straightforward
$P(X > k)$ Strictly greater pbinom(k, size, 0.5, lower.tail = FALSE) Does NOT include $k$
$P(X \geq k)$ Includes $k$ pbinom(k-1, size, 0.5, lower.tail = FALSE) THE TRAP: use $k - 1$
$P(X = k)$ Exact dbinom(k, size, 0.5) d not p

The critical one is the third row. lower.tail = FALSE gives strictly greater than. So for "at least $k$" (includes $k$), you subtract 1.

Example from HW1

Basketball player made 5 of 9 three-pointers. Test $H_0: p = 0.30$ vs $H_A: p > 0.30$. Need $P(X \geq 5)$ under $H_0$:

✗ Wrong

pbinom(5, 9, 0.3, lower.tail = FALSE)

This gives $P(X > 5) = P(X \geq 6)$. You're missing the case $X = 5$.

✓ Right

pbinom(4, 9, 0.3, lower.tail = FALSE)

This gives $P(X > 4) = P(X \geq 5)$. Includes 5.

$= 0.0988$

The rule: For $P(X \geq k)$, use pbinom(k - 1, size, prob, lower.tail = FALSE).
Say it out loud: "to include $k$, subtract 1 from the argument."
04 The discreteness problem: why you can't get $\alpha = 0.05$ exactly

With the t-test, you can get any significance level you want — 0.05, 0.01, 0.073, whatever. The $t$-distribution is continuous.

The sign test uses the Binomial distribution, which is discrete. The achievable tail probabilities are a fixed set determined by $n$. Let's see this with $n = 7$:

achievable_alphas.R
# Two-sided achievable significance levels for n = 7
# Reject when min(n+, n-) <= k

# k=0: 2 * pbinom(0, 7, 0.5) = 2 * 0.0078 = 0.0156
# k=1: 2 * pbinom(1, 7, 0.5) = 2 * 0.0625 = 0.1250
# k=2: 2 * pbinom(2, 7, 0.5) = 2 * 0.2266 = 0.4531

# Jumps: 0.016 → 0.125 → 0.453
# There's NO way to get 0.05!

The achievable levels jump from 0.016 straight to 0.125. There's no combination of outcomes that gives a tail probability of exactly 0.05. So the closest you can get is 0.016 (which is conservative — rejects less than 5% of the time).

Exam answer (from Practice Quiz 1, Q5b):
"The achievable alpha level of 0.015625 is the closest to 0.05 because the Binomial distribution is discrete — achievable significance levels jump from about 0.015625 to about 0.125, so we cannot get any closer to 0.05."

This means the sign test often has less power than the t-test — it's using a stricter threshold than it needs to (0.016 instead of 0.05).

05 Sign test vs t-test: power comparison

From HW1 Q3c — simulation with Normal($\mu = 0.5$, $\sigma = 1$), null $= 0$, $n = 30$:

TestPowerWhy
t-test≈ 0.75Uses actual magnitudes — knows 7.2 is further from 4 than 4.9 is
Sign test≈ 0.55Only counts signs — 7.2 and 4.9 both just count as "+"

The t-test has ~20 percentage points more power. Why?

The sign test throws away information. It only asks "above or below?" — it ignores HOW FAR above or below. The t-test uses the actual numbers (magnitudes), which gives it more information to work with, and therefore more power.

Think of it like grading: the t-test reads your essay, the sign test just checks if you wrote something.

So when should you use the sign test?

  • When data is heavily skewed and $n$ is small — the t-test's Type I Error is inflated, so its power advantage is unreliable
  • When you specifically want to test the median rather than the mean
  • When you have ordinal data (ranks, ratings) where magnitudes aren't meaningful

The sign test tests the _______ (mean/median). In one sentence, explain why the sign test and t-test can give different results on the same data.

Median. They can give different results because for skewed distributions the mean and median differ, so the tests are testing different parameters and can reach different conclusions about the same data.

Q3

$P(X \geq 7)$ where $X \sim \text{Binomial}(12, 0.3)$. Write the R code. Be careful with the off-by-one.

pbinom(6, size = 12, prob = 0.3, lower.tail = FALSE)
Use $k - 1 = 6$ to include 7. NOT pbinom(7, ...) which gives $P(X > 7) = P(X \geq 8)$.
Q4

Why does the sign test generally have less power than the t-test when data is normally distributed? Give two reasons.

1. The sign test throws away magnitude information — it only counts whether observations are above/below, not how far. The t-test uses actual values, giving it more information.
2. The sign test's discrete achievable alpha levels force it to use a stricter threshold than 0.05 (e.g., 0.016 for $n=7$), reducing its rejection rate even when $H_A$ is true.

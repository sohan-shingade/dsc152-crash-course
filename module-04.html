Part A Learn: Power Analysis
01 What is statistical power?

Think of a metal detector at the beach. Power is how good the detector is at finding buried metal. A weak detector misses coins. A strong detector finds them almost every time.

In statistics:

$$\text{Power} = P(\text{reject } H_0 \mid H_0 \text{ is false})$$

In words: power is the probability of correctly rejecting $H_0$ when $H_A$ is true.

Power connects to Type II Error:
Type II Error = failing to reject $H_0$ when it's false = $\beta$
Power = $1 - \beta$

High power = low chance of missing a real effect. 80% is the standard minimum (Lec 4).

What would power = 1.0 mean?

From Practice Quiz 1 Q2b: "The test would need to reject $H_0$ in every possible sample drawn under $H_A$, i.e. the test statistic always falls in the rejection region."

Power depends on a specific alternative. You can't compute power without specifying what the true parameter value actually is. For example: "power when $\mu = 50$" or "power when $\delta = 30$."

02 The 4 factors that control power

Four knobs you can turn. Memorize the direction of each:

↑
Sample size $n$ ↑ → Power ↑
More data = more information = easier to detect a real effect
↑
Effect size $\delta$ ↑ → Power ↑
Bigger true difference = easier to detect (finding a boulder vs a pebble)
↓
Standard deviation $\sigma$ ↓ → Power ↑
Less noise = signal stands out more
↑
Significance level $\alpha$ ↑ → Power ↑
Wider rejection region = easier to reject. But also more Type I Errors!
TRAP on Quiz 1 Q4d: Students often get $\sigma$ and $\alpha$ backwards.
Increase $\sigma$ = MORE noise = HARDER to detect = LESS power (not more!)
Decrease $\alpha$ from 0.05 to 0.01 = STRICTER threshold = HARDER to reject = LESS power
Memorize this: n↑ δ↑ σ↓ α↑ all increase power.
Think: "Need Detectable Signal, Accept risk" (N↑ D↑ S↓ A↑)
03 power.t.test() — the theory-based approach

When data is normally distributed, R can compute exact power using the non-central $t$-distribution. No simulation needed.

power_t_test.R
power.t.test(n = 10,          # sample size
             delta = 30,       # true mean - null value (effect size)
             sd = 10,          # population standard deviation
             sig.level = 0.05, # alpha
             type = "one.sample",
             alternative = "two.sided")
## power = 0.9999

Every argument matters. Let's walk through what each one means:

ArgumentWhat it isWatch out
nSample sizeThis is $n$, NOT degrees of freedom
delta$\mu_{\text{true}} - \mu_0$ (the shift)Always positive — it's the magnitude of the difference
sdPopulation $\sigma$NOT the standard error $\sigma/\sqrt{n}$
sig.level$\alpha$Default is 0.05, but quiz might use 0.10
type"one.sample"Must specify — default is "two.sample"
alternative"two.sided" or "one.sided"Must match the $H_A$ direction

Worked Example: Practice Quiz 1 Q2c (TSA wait times)

Context: $H_0: \mu = 20$, true $\mu = 50$, so $\delta = 30$. $\sigma = 10$, $n = 10$.

power.t.test(n = 10, delta = 30, sd = 10, sig.level = 0.05, type = "one.sample", alternative = "two.sided") ## power = 0.9999
Answer key interpretation (PQ1 Q2c): "If the true mean wait time is 50 minutes (30 minutes above the null value of 20), then with a sample size of 10 passengers, we would almost certainly detect this difference and correctly reject $H_0$."

One-sided example: Quiz 1 Q4b (delivery tips)

Context: $H_0: \mu = 4$, $H_A: \mu > 4$, true $\mu = 6$, so $\delta = 2$. $\sigma = 3$, $n = 30$.

             sig.level = 0.05,
             type = "one.sample",

Notice alternative = "one.sided" because the quiz specifies $H_A: \mu > 4$. Getting this wrong changes the power value.
04 Simulation-based power — when and how

Simulation gives you power for any distribution, not just Normal. The idea is dead simple:

The recipe:
1. Generate data from the true distribution (where $H_A$ is true)
2. Run the t-test on that data
3. Check if the p-value is below $\alpha$
4. Repeat 10,000 times
5. Power = proportion of times you rejected

The sim_power function (Practice Quiz 1 Q6a)

This exact code structure appears on exams. Every blank has been tested:

sim_power.R
sim_power <- function(n, delta, sd, alpha = 0.05, reps = 10000){
    count <- 0
    for(i in 1:reps){
        samp <- rnorm(n = n,
                      mean = 20 + delta,  # true mean = null + shift
                      sd   = sd)
        pval <- t.test(samp, mu = 20)$p.value
        if(pval < alpha){
            count <- count + 1
        }
    }
    return(count / reps)
}

Let's break down each piece that gets tested as blanks:

BlankAnswerWhy
rnorm(n = ___)nSample size is the function parameter
mean = ___20 + deltaTrue mean = null value + effect size (we simulate under $H_A$!)
sd = ___sdStandard deviation parameter
if(___ < ___)pval < alphaCount rejections: p-value below threshold
return(___)count / repsPower = fraction of rejections
Key difference from Type I Error simulation:
Type I Error sim: generate data where $H_0$ is TRUE (mean = 20)
Power sim: generate data where $H_A$ is TRUE (mean = 20 + delta)

Same loop structure, different mean. That's it.

Interpreting sim_power output (PQ1 Q6b)

If sim_power(n=5, delta=2, sd=1) returns ~0.91:

Answer key (PQ1 Q6b): "If the true average wait time is 22 minutes (2 minutes above the null value of 20), and the standard deviation is 1 minute, then with a sample size of 5 passengers we would correctly reject $H_0$ about 91% of the time."

One-sided version (Quiz 1 Q4a)

When $H_A$ is one-sided, the t.test call needs alternative:

    pval <- t.test(samp, mu = 4,
                   alternative = "greater")$p.value

        count <- count + 1
    }
}
count / 10000
05 When power.t.test() CANNOT be used

This is a guaranteed exam question. Quiz 1 Q4c tested it directly.

Scenario: data follows a right-skewed Gamma distribution, not Normal. Can you use power.t.test()?

Answer key (Quiz 1 Q4c): "No, power.t.test assumes the data follow a normal distribution. There is in fact no closed-form solution to the distribution of the t-Test test statistic if the data follow a gamma distribution, so there is no theory-based equivalent when the data are gamma distributed."

So what do you do instead? Simulate.

power_gamma.R
# Power when data is Gamma (Quiz 1 Q4c context)
count <- 0
for(i in 1:10000){
    samp <- rgamma(n = 30, shape = 1.2, scale = (6 / 1.2))
    pval <- t.test(samp, mu = 4,
                   alternative = "greater")$p.value
    if(pval < 0.05){
        count <- count + 1
    }
}
count / 10000

Notice: the only thing that changed is rnorm(...) became rgamma(...). The rest of the simulation is identical. That's the beauty of simulation — swap the distribution, keep everything else.

power.t.test()

  • Data must be Normal
  • Exact answer (no randomness)
  • One line of code
  • Uses non-central $t$-distribution

Simulation

  • Works for ANY distribution
  • Approximate (varies by run)
  • Requires a loop
  • Must know how to generate data
When to use which:
• Normal data + t-test → power.t.test() works, simulation also works
• Non-normal data (Gamma, skewed, etc.) → simulation ONLY
• If the exam says "theory-based equivalent" — that means power.t.test()

08 Exam Q Type: "Which factors increase power?" + "Can power.t.test be used?"

Real exam question (Quiz 1 Q4d):

Which of the following changes, applied individually, would increase the statistical power of the test? Select all that apply.

◼ Increase the sample size $n$
◼ Increase the effect size $\delta$
◻ Increase the population standard deviation $\sigma$
◻ Decrease the significance level $\alpha$ from 0.05 to 0.01
◻ None of the above

Why the wrong answers are wrong:

  • Increase $\sigma$: More noise makes it HARDER to see the signal. Power DECREASES.
  • Decrease $\alpha$: Stricter threshold (0.01 vs 0.05) makes it HARDER to reject. The rejection region shrinks. Power DECREASES.

Real exam question (Quiz 1 Q4c):

Now consider simulation with rgamma(n = 30, shape = 1.2, scale = (6 / 1.2)). Can a theory-based equivalent of the power estimated by this simulation be calculated? If so, write R code. If not, explain why.

Sample size isn't the issue — the distribution is.

Full marks

"No, power.t.test assumes the data follow a normal distribution. There is in fact no closed-form solution to the distribution of the t-Test test statistic if the data follow a gamma distribution, so there is no theory-based equivalent when the data are gamma distributed."

09 4-Question Quiz
Q1

Define statistical power in one sentence using proper terminology. Then: if power = 0.74 for a test with $n = 25$, $\delta = 3$, $\sigma = 8$, and $\alpha = 0.05$, interpret what 0.74 means in plain English.

Definition: Statistical power is the probability of correctly rejecting $H_0$ when $H_0$ is actually false (i.e., when $H_A$ is true).

Interpretation: If the true mean is 3 units away from the null value and $\sigma = 8$, then with $n = 25$ at $\alpha = 0.05$, we would correctly reject $H_0$ about 74% of the time. There's a 26% chance we'd miss the effect (Type II Error).
Q2

Write the power.t.test() call for: one-sample t-test, $H_0: \mu = 100$, $H_A: \mu \neq 100$, true $\mu = 108$, $\sigma = 15$, $n = 40$, $\alpha = 0.01$.

power.t.test(n = 40, delta = 8, sd = 15,
             sig.level = 0.01,
             type = "one.sample",
             alternative = "two.sided")
Key: delta = 108 - 100 = 8, sig.level = 0.01 (not 0.05!), "two.sided" because $H_A: \mu \neq 100$.

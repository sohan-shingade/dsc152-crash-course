Part A Learn: Categorical Predictors & Partial F-Test
01 How R handles categorical predictors — dummy variables & reference category

When you include a categorical variable in lm(), R doesn't plug in "rural" or "urban" as numbers. Instead, it creates dummy variables — binary 0/1 columns for each level except one.

The level that gets left out is the reference category. By default, R picks the level that comes first alphabetically.

Reference category rule: R sorts the levels of a factor alphabetically and uses the first one as the reference. For a variable with levels "rural", "suburban", "urban" → the reference is "rural" because r comes before s and u.

For a categorical variable loc_type with 3 levels (rural, suburban, urban), R creates 2 dummy variables:

Observationloc_typesuburbanloc_typeurban
rural00
suburban10
urban01
General rule: A categorical variable with $k$ levels produces $k - 1$ dummy variables. The reference category is identified by having all dummies equal to 0.
02 Interpreting categorical coefficients — "compared to [reference]..."

Every coefficient for a dummy variable tells you the difference from the reference category, holding other predictors constant.

Interpretation template: "Compared to [reference category], [this level] is associated with a $\hat{\beta}$ [increase/decrease] in Y, holding other predictors constant."

Example from Practice Quiz 2 Q2c:

Model: lm(sal ~ emp_n + loc_type, data = ds_jobs) Reference category: "rural" (alphabetically first) Coefficient for loc_typesuburban: -8200
Interpretation: "Suburban jobs are associated with a $8,200 lower average starting salary than rural jobs, among companies with the same number of employees."

Loses points

"The coefficient for suburban is -8200."

Just restating the number. You need to say compared to WHAT, and hold other variables constant.

Full marks

"Compared to rural jobs, suburban jobs are associated with a $8,200 lower average starting salary, among companies with the same number of employees."
Watch out: The coefficient name in R output looks like loc_typesuburban (variable name glued to level name, no space). Don't let this confuse you — it just means the dummy variable for the "suburban" level of loc_type.
03 R code — fitting models with categorical predictors

The beauty of R: you don't need to manually create dummy variables. Just include the categorical column in the formula, and R handles everything behind the scenes.

categorical_lm.R
# Numeric + categorical predictor
model <- lm(sal ~ emp_n + loc_type, data = ds_jobs)
summary(model)

R automatically creates dummies for loc_type. In the summary() output, you'll see separate rows for each non-reference level:

Coefficients: Estimate Std. Error t value Pr(>|t|) (Intercept) 85000 ... ... ... emp_n 3.2 ... ... ... loc_typesuburban -8200 ... ... 0.043 loc_typeurban 12500 ... ... 0.001
Reading the output: There's no row for loc_typerural because rural is the reference. The intercept already represents a rural observation (when emp_n = 0). Each dummy coefficient adjusts from there.

The predicted equation is:

$$\hat{y} = 85000 + 3.2 \cdot \text{emp\_n} - 8200 \cdot \text{suburban} + 12500 \cdot \text{urban}$$
04 The Partial F-Test — testing a GROUP of coefficients at once

Here's the problem: a categorical variable with $k$ levels creates $k - 1$ dummy variables, each with its own coefficient. You can't just look at individual t-tests — you need to test whether all those coefficients are zero simultaneously.

Key idea: The partial F-test asks: "Does this categorical variable, as a whole, significantly improve the model?" It tests whether ALL the coefficients for that variable are zero at the same time.

$$H_0: \beta_1 = \beta_2 = \cdots = \beta_{k-1} = 0$$ $$H_A: \text{at least one } \beta_j \neq 0$$

You do this by comparing two nested models with anova():

partial_f_test.R
# Null model: WITHOUT the categorical variable
null_mod <- lm(sal ~ emp_n, data = ds_jobs)

# Full model: WITH the categorical variable
full_mod <- lm(sal ~ emp_n + loc_type, data = ds_jobs)

# Partial F-test
anova(null_mod, full_mod)
Order matters: The null (smaller) model goes FIRST in anova(). The full (larger) model goes second. Think of it as: anova(without, with).

The F-statistic measures how much the full model reduces residual error compared to the null model:

$$F = \frac{(RSS_{\text{null}} - RSS_{\text{full}}) / p}{RSS_{\text{full}} / (n - k)}$$

Where $p$ = number of extra coefficients in the full model, $n$ = sample size, $k$ = total coefficients in the full model.

05 Reading anova() output — F statistic, p-value, interpretation

Here's what anova() output looks like and what each piece means:

Analysis of Variance Table Model 1: sal ~ emp_n Model 2: sal ~ emp_n + loc_type Res.Df RSS Df Sum of Sq F Pr(>F) 1 198 4500000 2 196 3800000 2 700000 18.03 4.2e-08 ***
ColumnMeaning
Res.DfResidual degrees of freedom for each model
RSSResidual sum of squares — lower is better
DfDifference in number of parameters (= number of new dummies added)
Sum of SqReduction in RSS from adding the variable
FThe F-statistic — bigger means stronger evidence against $H_0$

Interpretation template: "Since the p-value [value] is [less/greater] than $\alpha = 0.05$, we [reject/fail to reject] $H_0$. There is [statistically significant / not statistically significant] evidence that at least one [variable] group differs in mean [outcome], adjusting for [other predictors]."

Example from PQ2 Q3c:

"Reject $H_0$; there is statistically significant evidence that at least one modality group differs in mean salary, adjusting for rent_1br."
Why not just use individual t-tests? With multiple dummies, running separate t-tests inflates Type I error (multiple comparisons problem). The F-test handles all levels in one shot, controlling the overall error rate.

In the model lm(sal ~ emp_n + loc_type, data = ds_jobs), where loc_type has levels "rural", "suburban", and "urban": What is the reference category? Interpret the coefficient for loc_typesuburban = -8200.

Loses points

Missing: compared to what? Holding what constant?

"The reference category is rural (alphabetically first among levels of loc_type). The coefficient of -8,200 for loc_typesuburban means suburban jobs are associated with a $8,200 lower average starting salary than rural jobs, among companies with the same number of employees."
Checklist for categorical coefficient interpretations:
1. Name the reference category and WHY it's the reference (alphabetically first)
2. State the comparison: "[this level] compared to [reference]"
3. Give the direction and magnitude: "associated with a [$X] [higher/lower]..."
4. Hold other variables constant: "among [units] with the same [other predictors]"
Your Turn

A model lm(gpa ~ hours_studied + major, data = students) has majors "Biology", "CS", "Math". The coefficient for majorCS is 0.35. What is the reference category? Interpret this coefficient.

The reference category is Biology (alphabetically first). The coefficient of 0.35 means that CS majors are associated with a 0.35 higher GPA than Biology majors, among students with the same number of hours studied.
07 Exam Q Type: "Write the anova() code for a partial F-test"

Real exam question (PQ2 Q3a):

You want to test whether modality (a categorical variable) is a significant predictor of sal (salary), after adjusting for rent_1br. Write the R code for the partial F-test.

Answer key (PQ2 Q3a):
partial_f_pq2.R
# Step 1: Null model (WITHOUT modality)
null_mod <- lm(sal ~ rent_1br, data = ds_jobs)

# Step 2: Full model (WITH modality)
full_mod <- lm(sal ~ modality + rent_1br, data = ds_jobs)

# Step 3: Compare them
anova(null_mod, full_mod)
Recipe for partial F-test code (3 lines):
1. null_mod <- lm(y ~ [other predictors], data = df) — everything EXCEPT the variable you're testing
2. full_mod <- lm(y ~ [test variable] + [other predictors], data = df) — add the variable back in
3. anova(null_mod, full_mod) — null first, full second
Common mistake: Putting the full model first in anova(full_mod, null_mod). The smaller (null) model must come first.
Your Turn

You want to test whether region (a categorical variable with levels East, North, South, West) is significant in predicting price, after adjusting for sqft. Data is in houses. Write the 3 lines of R code.

null_mod <- lm(price ~ sqft, data = houses)
full_mod <- lm(price ~ region + sqft, data = houses)

The partial F-test for modality returns a p-value of 0.002. At $\alpha = 0.05$, what do you conclude?

Loses points

"The p-value is less than 0.05 so we reject H0."

What does rejecting mean here? Need to connect back to what was tested.

Full marks

"Reject $H_0$; there is statistically significant evidence that at least one modality group differs in mean salary, adjusting for rent_1br."
F-test interpretation template:
"[Reject / Fail to reject] $H_0$; there is [statistically significant / not statistically significant] evidence that at least one [categorical variable] group differs in mean [outcome], adjusting for [other predictors]."
Key phrasing: "at least one ... group differs." The F-test tells you SOMETHING is different, but not WHICH groups differ. For that, you'd need post-hoc tests (not covered in DSC 152).

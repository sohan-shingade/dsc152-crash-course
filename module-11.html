Part A Learn: Interaction Terms
01 What interaction means — parallel vs non-parallel lines

An interaction means the effect of one predictor ($X_1$) on the outcome ($Y$) depends on the level of another predictor ($X_2$).

No Interaction

  • Lines for each group are parallel
  • The effect of $X_1$ is the same regardless of $X_2$
  • Groups differ only in intercept

Interaction Present

  • Lines for each group are NOT parallel
  • The effect of $X_1$ differs depending on $X_2$
  • Groups differ in both intercept AND slope
Key visual test: If you plot fitted lines for each group and they cross or diverge, there's an interaction. Parallel lines = no interaction.

Think of it this way: without interaction, going to the gym improves your mental health by the same amount whether or not you also meditate. With interaction, the gym benefit is different depending on whether you meditate.

02 R syntax for interaction — the * and : operators

In R, the * operator in a formula adds both main effects AND the interaction:

interaction_syntax.R
# These two are IDENTICAL:
mod_int <- lm(y ~ x1 * x2, data = df)
mod_int <- lm(y ~ x1 + x2 + x1:x2, data = df)
OperatorWhat it addsExample
*Main effects + interactionx1 * x2 = x1 + x2 + x1:x2
:Only the interaction termx1:x2 = just the product term
+Only main effects (no interaction)x1 + x2 = additive model
Common mistake: Writing lm(y ~ x1:x2, data = df) without the main effects. Always use * or explicitly include both main effects with + when you add :. Dropping main effects gives misleading results.

03 Interpreting interaction coefficients — the ADDITIONAL effect

The interaction coefficient is not a standalone effect. It tells you how much the slope changes when you move from the reference group to the other group.

Consider a model with one numeric variable ($x_1$) and one binary variable ($x_2$ = TRUE/FALSE):

$$\hat{y} = \hat{\beta}_0 + \hat{\beta}_1 x_1 + \hat{\beta}_2 x_{2\text{TRUE}} + \hat{\beta}_3 (x_1 \cdot x_{2\text{TRUE}})$$
CoefficientInterpretation
$\hat{\beta}_1$Slope of $x_1$ for the reference group ($x_2$ = FALSE)
$\hat{\beta}_2$Difference in intercept between TRUE and FALSE groups when $x_1 = 0$
$\hat{\beta}_3$How much the slope of $x_1$ changes for the TRUE group vs the FALSE group
Think of $\hat{\beta}_3$ as: "The slope of $x_1$ is $\hat{\beta}_3$ more for the TRUE group than for the FALSE group." So the TRUE group's slope is $\hat{\beta}_1 + \hat{\beta}_3$.
PQ2 Q4 coefficients:
Intercept = 74,000  |  hi_rentTRUE = 21,000  |  modalityremote = 6,500  |  hi_rentTRUE:modalityremote = 9,800

The interaction (9,800) means: "The salary boost from being in a high-rent city is $9,800 larger for remote jobs than for in-person jobs."
04 Writing fitted equations for subgroups — plug in dummy values

To get the fitted equation for a specific subgroup, plug in the dummy variable values and simplify.

General model:

$$\hat{y} = \beta_0 + \beta_1 x + \beta_2 \cdot \text{group} + \beta_3 \cdot x \cdot \text{group}$$

group = 0 (reference)

$\hat{y} = \beta_0 + \beta_1 x + \beta_2(0) + \beta_3 x(0)$

$\hat{y} = \beta_0 + \beta_1 x$

group = 1

$\hat{y} = \beta_0 + \beta_1 x + \beta_2(1) + \beta_3 x(1)$

$\hat{y} = (\beta_0 + \beta_2) + (\beta_1 + \beta_3) x$

PQ2 Q4b — Fitted equation for remote jobs:
Full model: $\widehat{\text{sal}} = 74000 + 21000 \cdot \text{hi\_rent} + 6500 \cdot \text{remote} + 9800 \cdot \text{hi\_rent} \cdot \text{remote}$

Plug in remote = 1:
$\widehat{\text{sal}} = (74000 + 6500) + (21000 + 9800) \cdot \text{hi\_rent}$
$\widehat{\text{sal}} = 80500 + 30800 \cdot \text{hi\_rentTRUE}$
New intercept: 74000 + 6500 = 80500 New slope: 21000 + 9800 = 30800 Remote jobs: sal_hat = 80500 + 30800 * hi_rentTRUE In-person: sal_hat = 74000 + 21000 * hi_rentTRUE
Recipe: For the non-reference group, add the dummy coefficient to the intercept, and add the interaction coefficient to the slope.
05 When to use interaction — and why you need partial F-tests

Use an interaction when you suspect the relationship between $X$ and $Y$ differs across groups. Signs to look for:

  • Non-parallel lines in a scatterplot colored by group
  • Theory suggests groups respond differently
  • The interaction coefficient is significant
Why you can't just look at individual p-values (PQ2 Q4g):
The p-value for modalityremote (0.038) only tests the impact of modality in low-rent locations specifically (when hi_rent = FALSE, the reference level). We probably care about the impact of modality in ALL locations. That single p-value doesn't capture the full picture.

Instead, use a partial F-test (via anova()) to test whether modality matters overall:

partial_f_test.R
# PQ2 Q4h: Does modality matter AT ALL?
null_mod2 <- lm(sal ~ hi_rent, data = ds_jobs_2)
full_mod2 <- lm(sal ~ hi_rent * modality, data = ds_jobs_2)
anova(null_mod2, full_mod2)
What the F-test tests: $H_0: \beta_{\text{remote}} = \beta_{\text{hi\_rent:remote}} = 0$ simultaneously. This asks: "Is modality irrelevant everywhere?" — which is the actual question we care about.

08 Exam Q Type: Why not just use the p-value? / Writing the correct F-test

PQ2 Q4g: The p-value for modalityremote is 0.038. Can we use this alone to decide if modality matters?

Loses points

"p = 0.038 < 0.05 so modality is significant."

Ignores that this p-value only tests modality at the reference level of hi_rent.

Full marks

"The p-value of 0.038 is for the impact of modality only in low-rent locations, and we probably care about the impact of modality in all locations."

PQ2 Q4h — The correct approach: Use a partial F-test.

f_test_modality.R
# Null model: modality doesn't matter at all
null_mod2 <- lm(sal ~ hi_rent, data = ds_jobs_2)

# Full model: modality matters (main effect + interaction)
full_mod2 <- lm(sal ~ hi_rent * modality, data = ds_jobs_2)

# F-test: H0: beta_remote = beta_hi_rent:remote = 0
anova(null_mod2, full_mod2)
Rule of thumb: When a variable appears in BOTH a main effect and an interaction, use anova() to test whether it matters overall. Don't rely on individual summary() p-values.
Part C Quiz: Interaction Terms
09 4-Question Quiz
Q1

A model is fit: lm(gpa ~ hours_studied * tutor, data = students) where tutor is "yes" or "no". The output shows: Intercept = 2.1, hours_studied = 0.15, tutoryes = 0.3, hours_studied:tutoryes = 0.08. Interpret the interaction coefficient (0.08) in context.

The slope of hours_studied is 0.08 higher for students with a tutor than for students without a tutor. In other words, each additional hour of study is associated with a 0.08 greater increase in GPA for tutored students compared to non-tutored students. The tutored group's slope is 0.15 + 0.08 = 0.23 per hour, while the non-tutored group's slope is 0.15 per hour.
Q2

Using the same model from Q1, write the fitted equation for students who have a tutor (tutor = "yes").

Plug in tutoryes = 1:
$\widehat{\text{gpa}} = (2.1 + 0.3) + (0.15 + 0.08) \cdot \text{hours\_studied}$
$\widehat{\text{gpa}} = 2.4 + 0.23 \cdot \text{hours\_studied}$
Q3

In the salary model lm(sal ~ hi_rent * modality), the coefficient for hi_rentTRUE is 21,000. A student says: "Being in a high-rent city is associated with $21,000 higher salary for all jobs." What is wrong with this interpretation?

The $21,000 coefficient applies only to in-person jobs (the reference level of modality). Because there is an interaction term, the effect of hi_rent differs by modality. For remote jobs, the effect of being in a high-rent city is 21,000 + 9,800 = $30,800. In an interaction model, each main effect coefficient only holds when the other variable is at its reference level.
Q4

You fit lm(sal ~ hi_rent * modality) and the p-value for modalityremote in the summary() output is 0.12. A classmate says: "Modality is not significant, so we can drop it from the model." Explain why this reasoning is flawed and what they should do instead.

The p-value of 0.12 only tests whether modality matters at the reference level of hi_rent (i.e., in low-rent cities only). Modality could still be important in high-rent cities via the interaction term. To properly test whether modality matters overall, use a partial F-test:

null_mod <- lm(sal ~ hi_rent, data = ds_jobs_2)
full_mod <- lm(sal ~ hi_rent * modality, data = ds_jobs_2)
anova(null_mod, full_mod)

This tests $H_0: \beta_{\text{remote}} = \beta_{\text{hi\_rent:remote}} = 0$ simultaneously.

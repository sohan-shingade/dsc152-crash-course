Part A Learn: Logistic Regression
01 Why logistic regression — linear regression breaks for binary outcomes

When your outcome is binary (0/1, yes/no, admitted/rejected), linear regression doesn't work.

The problem with linear regression for binary outcomes: A linear model $\hat{y} = \beta_0 + \beta_1 x$ can predict values outside 0 and 1. If you're predicting probability of admission, you could get $\hat{y} = -0.3$ or $\hat{y} = 1.4$. Those aren't valid probabilities.

Linear regression

  • Predictions can go below 0 or above 1
  • Assumes normally distributed errors
  • Not appropriate for binary Y

Logistic regression

  • Predictions always between 0 and 1
  • Models log-odds as linear in X
  • Designed for binary Y

The fix: instead of modeling $p$ directly, model the log-odds of $p$. That's what logistic regression does.

02 The logistic regression model — log-odds = linear function

The core equation of logistic regression:

$$\log\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 x$$

where $p = P(Y = 1)$ is the probability of the outcome occurring.

PieceNameWhat it is
$p$Probability$P(Y=1)$, between 0 and 1
$\frac{p}{1-p}$OddsBetween 0 and $\infty$
$\log\left(\frac{p}{1-p}\right)$Log-odds (logit)Between $-\infty$ and $+\infty$
Why log-odds? Probability is bounded [0, 1]. Odds are bounded [0, $\infty$). Log-odds range over all real numbers $(-\infty, +\infty)$ — perfect for the right-hand side of a linear equation, which can also be any real number. The logit transformation maps probabilities to an unrestricted scale.
From Practice Quiz 3 Q4a: The fitted logistic regression model is written as:
$\log\left(\frac{\hat{p}}{1-\hat{p}}\right) = \hat{\beta}_0 + \hat{\beta}_1 x_1 + \hat{\beta}_2 x_2 + \ldots$

where $\hat{p}$ is the estimated probability of the outcome.
03 R code — glm() with family = binomial

In R, logistic regression uses glm(), not lm().

logistic_regression.R
# Logistic regression syntax
model_logit <- glm(admitted ~ gre + gpa,
                   data = grad,
                   family = "binomial")
summary(model_logit)
Three things that differ from linear regression:
1. Use glm() instead of lm()
2. Add family = "binomial" (or family = binomial)
3. The output shows z values instead of t values
From Practice Quiz 3 Q4: The exact syntax from the quiz is:
model_logit <- glm(admitted ~ gre + gpa, data = grad, family = "binomial")

If you forget family = "binomial", R will fit a regular linear model instead. That's a common exam mistake.
04 Interpreting coefficients — log-odds and odds ratios

This is the most commonly tested concept. Coefficients in logistic regression are on the log-odds scale.

$$\beta_1 = \text{change in log-odds of } Y=1 \text{ for a 1-unit increase in } X$$

But nobody thinks in log-odds. To make it interpretable, exponentiate to get the odds ratio:

$$e^{\beta_1} = \text{multiplicative change in odds for a 1-unit increase in } X$$
From Practice Quiz 3 Q4b (exact solution wording):
With $\hat{\beta}_{\text{gre}} = 0.08$: "For any small value $x$, $e^x \approx 1 + x$. Thus, $e^{0.08} \approx 1.08$. This indicates that each additional point on the GRE score is associated with a multiplicative increase of approximately 1.08 in the estimated odds of admission (roughly an 8% increase in the odds of admission), holding GPA constant."

Loses points

"A 1-point increase in GRE increases the probability of admission by 0.08."

Wrong! 0.08 is the change in LOG-ODDS, not probability. And it's a multiplicative effect on odds, not additive on probability.

Full marks

"A 1-point increase in GRE is associated with a multiplicative increase of $e^{0.08} \approx 1.08$ in the odds of admission, holding GPA constant."
Interpretation checklist:
1. Say "log-odds" or convert to odds ratio with $e^{\beta}$
2. Say "1-unit increase in X"
3. Say "holding other variables constant"
4. Say "associated with" (not "causes") unless it's an experiment
05 Predicted probability — going from coefficients to $\hat{p}$

Given coefficients, you can compute the predicted probability for any values of X. Here's the step-by-step process:

Step 1: Compute log-odds log-odds = β₀ + β₁x₁ + β₂x₂ Step 2: Convert to odds odds = e^(log-odds) Step 3: Convert to probability p = odds / (1 + odds)

Or in one formula:

$$\hat{p} = \frac{e^{\beta_0 + \beta_1 x_1 + \beta_2 x_2}}{1 + e^{\beta_0 + \beta_1 x_1 + \beta_2 x_2}}$$
From Practice Quiz 3 Q4e: "An applicant has a GRE score of 160 and a GPA of 3.5. Write an expression for their estimated probability of admission."

Solution:
Log-odds = $-14.82 + 0.08(160) + 0.77(3.5)$
$\hat{p} = \frac{e^{-14.82 + 0.08(160) + 0.77(3.5)}}{1 + e^{-14.82 + 0.08(160) + 0.77(3.5)}}$
You don't always need the final number. On a no-calculator exam, writing the expression is usually full credit. Just make sure you plug in correctly and show the steps: log-odds → odds → probability.
06 Confidence intervals for odds ratios — exp(confint())

The summary() output gives CIs on the log-odds scale. To get CIs for the odds ratio, exponentiate.

or_confidence_interval.R
# CI for odds ratio
exp(confint(model_logit))
From Practice Quiz 3 Q4c: "A 95% confidence interval for $\beta_{\text{gre}}$ on the log-odds scale is (0.06, 0.10). Give the corresponding 95% CI for the odds ratio."

Solution: Exponentiate both endpoints: $(e^{0.06},\; e^{0.10}) \approx (1.06,\; 1.10)$.
"We are 95% confident that a 1-point increase in GRE score multiplies the odds of admission by a factor between approximately 1.06 and 1.10."
Key idea: If the log-odds CI is $(L, U)$, the odds ratio CI is $(e^L, e^U)$. Just exponentiate both endpoints. If the OR CI contains 1, the predictor is not significant at that level.
07 Three conditions for logistic regression

Just like linear regression has conditions (LINE), logistic regression has three conditions you need to check.

From Practice Quiz 3 Q4d (exact solution):
#ConditionHow to check (grad admissions context)
1Binary outcomeadmitted takes only values 0 and 1. Satisfied by construction.
2Independence of observationsEach row is a different applicant. Approximately satisfied if applicants are not clustered (e.g., not many pairs from the same program sharing advisors).
3Linearity on the log-odds scaleThe log-odds of admission should be approximately linear in gre and gpa. Check via a deviance residual plot (fitted values vs. deviance residuals), looking for a loess smoother that approximately tracks the horizontal zero line.
Common mistake: Saying "normality" is a condition. Logistic regression does not require normally distributed data. That's a linear regression condition. The outcome is binary — it can't be normal!
check_linearity.R
# Checking linearity condition via deviance residuals
fitted_vals <- model_logit$fitted.values
dev_resids  <- residuals(model_logit, type = "deviance")

ggplot(data.frame(fitted = fitted_vals, resid = dev_resids),
       aes(x = fitted, y = resid)) +
  geom_point() +
  geom_smooth(method = "loess", se = FALSE) +
  geom_hline(yintercept = 0, linetype = "dashed")

09 Exam Q Type: "Compute the predicted probability for x = ..."

You'll be given specific values and asked to write an expression (or compute) the predicted probability.

Using the model $\log\left(\frac{\hat{p}}{1-\hat{p}}\right) = -14.82 + 0.08 \cdot \text{gre} + 0.77 \cdot \text{gpa}$, find the estimated probability of admission for an applicant with GRE = 160 and GPA = 3.5.

Step 1: log-odds = -14.82 + 0.08(160) + 0.77(3.5) = -14.82 + 12.80 + 2.695 = 0.675 Step 2: odds = e^0.675 Step 3: p̂ = e^0.675 / (1 + e^0.675)
On a no-calculator exam: You typically just need to write the expression correctly. Show three clear steps: (1) plug in to get log-odds, (2) exponentiate to get odds, (3) convert to probability. You won't be expected to evaluate $e^{0.675}$ by hand.
Your Turn

Same model. Compute the predicted probability expression for an applicant with GRE = 155 and GPA = 3.8.

Log-odds = $-14.82 + 0.08(155) + 0.77(3.8) = -14.82 + 12.40 + 2.926 = 0.506$

$\hat{p} = \frac{e^{0.506}}{1 + e^{0.506}}$

(If you want to approximate: $e^{0.5} \approx 1.65$, so $\hat{p} \approx 1.65 / 2.65 \approx 0.62$)

Your Turn

The 95% CI for $\beta_{\text{gpa}}$ is (0.40, 1.14) on the log-odds scale. What is the 95% CI for the odds ratio associated with a 1-point increase in GPA?

Exponentiate both endpoints: $(e^{0.40}, e^{1.14}) \approx (1.49, 3.13)$.

Part C Quiz: Logistic Regression
11 4-Question Quiz
Q1

A researcher wants to predict whether a customer will churn (yes/no) based on their monthly bill. They fit a linear regression model $\hat{y} = 1.2 - 0.005 \cdot \text{bill}$. For a customer with a $300 monthly bill, the model predicts $\hat{y} = 1.2 - 0.005(300) = -0.3$. What is wrong with this approach, and what should they use instead?

The predicted value $\hat{y} = -0.3$ is outside the valid range of [0, 1] for a probability. Linear regression is not appropriate for binary outcomes because it can produce predictions below 0 or above 1. They should use logistic regression, which models the log-odds as a linear function of the predictors, ensuring predicted probabilities always fall between 0 and 1. In R: glm(churn ~ bill, data = df, family = "binomial").
Q2

A logistic regression predicting loan default from credit score gives $\hat{\beta}_1 = -0.03$. Interpret this coefficient in two ways: (a) on the log-odds scale, and (b) on the odds ratio scale.

(a) Log-odds: A 1-point increase in credit score is associated with a decrease of 0.03 in the log-odds of loan default.

(b) Odds ratio: $e^{-0.03} \approx 0.97$. A 1-point increase in credit score is associated with multiplying the odds of default by approximately 0.97, i.e., roughly a 3% decrease in the odds of default.
Q3

Given the model $\log\left(\frac{\hat{p}}{1-\hat{p}}\right) = -3.5 + 0.02 \cdot \text{score} + 1.2 \cdot \text{interview}$, write an expression for the predicted probability that a candidate with score = 150 and interview = 1 gets hired.

Step 1: Log-odds = $-3.5 + 0.02(150) + 1.2(1) = -3.5 + 3.0 + 1.2 = 0.7$

Step 2: Odds = $e^{0.7}$

Step 3: $\hat{p} = \frac{e^{0.7}}{1 + e^{0.7}}$

(Approximation: $e^{0.7} \approx 2.01$, so $\hat{p} \approx 2.01 / 3.01 \approx 0.67$)
Q4

Write one line of R code to fit a logistic regression predicting survived from age and fare using a data frame called titanic. Then write one line of R code to obtain 95% confidence intervals for the odds ratios.

model <- glm(survived ~ age + fare, data = titanic, family = "binomial")

exp(confint(model))

Remember: glm not lm, family = "binomial" is required, and exp(confint(...)) converts from log-odds CI to odds ratio CI.

Part A Learn: ACF/PACF & AR Models (Lec 18)
01 What makes time series special — observations are NOT independent

In all the regression we've done so far, we assumed the Independence condition in LINE: each observation is independent of the others. Time series data violates this.

Today's value depends on yesterday's. A basketball player who scored 40 points last game might get double-teamed tonight, affecting his score. Stock prices carry momentum from one day to the next. These are not independent observations.

LINE conditions for regression:
Linearity • Independence • Normality of $\epsilon_i$ • Equal variance (homoskedasticity)

Time series data violates the I — so we need special models that account for this dependency.
Why does this matter? If you ignore time dependency and just use lm(), your p-values will be wrong. The Independence assumption is baked into how standard errors are calculated. Violate it, and the whole inference breaks down.
02 Key terminology: $Y_t$, $\eta_t$, and the difference between them

Two symbols you need to keep straight:

$Y_t$ — the observation

  • The actual value at time $t$
  • e.g., points scored in game $t$
  • NOT independent across time
  • $Y_t$ depends on $Y_{t-1}$, $Y_{t-2}$, etc.

$\eta_t$ — the shock / innovation

  • The random noise entering at time $t$
  • e.g., a random hot or cold streak
  • Independent across time
  • Each $\eta_t \sim N(0, \sigma^2)$ on its own
The key distinction: The shocks $\eta_t$ (also written $\epsilon_t$ in some notation) are independent of each other. But the observations $Y_t$ are not independent, because each $Y_t$ is built from previous $Y$ values plus the current shock. The dependency between observations is what makes this time series data.

Think of it like dominoes: each shock $\eta_t$ is an independent push, but the observation $Y_t$ reflects all the previous pushes that haven't died out yet.

03 ACF — Autocorrelation Function

The ACF measures the correlation between $Y_t$ and $Y_{t-k}$ at different lags $k$.

  • Lag 1: correlation between $Y_t$ and $Y_{t-1}$ (consecutive observations)
  • Lag 2: correlation between $Y_t$ and $Y_{t-2}$ (two steps apart)
  • Lag $k$: correlation between $Y_t$ and $Y_{t-k}$
acf_example.R
# Plot the ACF
acf(data)

# The plot shows bars at each lag
# Bars outside the blue dashed lines = significant autocorrelation
How to read an ACF plot: The x-axis is the lag number, the y-axis is the correlation. The blue dashed lines mark the significance threshold. If a bar extends past those lines, there is statistically significant autocorrelation at that lag.

For an AR(p) process, the ACF decays gradually (tails off slowly). It doesn't cut off sharply — it just gets smaller and smaller. This is because in an AR model, impacts linger: $Y_t$ depends on $Y_{t-1}$, which depends on $Y_{t-2}$, and so on. So even distant lags show some correlation.

04 PACF — Partial Autocorrelation Function

The PACF measures the correlation between $Y_t$ and $Y_{t-k}$ after removing the effect of all intermediate lags ($Y_{t-1}, Y_{t-2}, \ldots, Y_{t-k+1}$).

Why "partial"? The regular ACF at lag 2 includes both the direct effect of $Y_{t-2}$ on $Y_t$ AND the indirect effect through $Y_{t-1}$. The PACF strips out the indirect part, showing only the direct connection.
pacf_example.R
# Plot the PACF
pacf(data)

# For AR(p): PACF cuts off sharply after lag p
# This is how you identify the order of an AR model
The PACF is the key tool for identifying AR order:
• PACF cuts off after lag 1 → AR(1)
• PACF cuts off after lag 2 → AR(2)
• PACF cuts off after lag $p$ → AR($p$)

"Cuts off" means the bars after lag $p$ are all within the blue dashed significance lines.
ModelACF patternPACF pattern
AR($p$)Decays gradually (tails off)Cuts off after lag $p$
MA($q$)Cuts off after lag $q$Decays gradually (tails off)

Notice the symmetry: ACF and PACF swap roles for AR vs MA models. This pattern is how you tell them apart.

05 AR($p$) model — the equation and fitting in R

The AR($p$) (autoregressive of order $p$) model says each observation is a linear combination of the previous $p$ observations plus noise:

$$Y_t = c + \phi_1 Y_{t-1} + \phi_2 Y_{t-2} + \cdots + \phi_p Y_{t-p} + \eta_t$$

where $c$ is a drift/intercept constant, each $\phi_i$ is a coefficient, and $\eta_t \sim N(0, \sigma^2)$ independently.

AR(1)

$Y_t = c + \phi_1 Y_{t-1} + \eta_t$

Each value depends on the previous one plus noise. Impacts linger because $Y_{t-1}$ depended on $Y_{t-2}$, and so on.

AR(2)

$Y_t = c + \phi_1 Y_{t-1} + \phi_2 Y_{t-2} + \eta_t$

Each value depends on the previous two plus noise. Even longer memory of the past.

Fitting in R

fit_ar.R
library(forecast)

# Fit an AR(1) model
model <- Arima(data, order = c(1, 0, 0))

# Fit an AR(2) model
model <- Arima(data, order = c(2, 0, 0))

# order = c(p, d, q)
#   p = AR order (autoregressive)
#   d = differencing (we don't use this)
#   q = MA order (moving average)
Note the capital A! It's Arima() from the forecast package, not arima() (base R). The capital-A version is what we use in this course.
Key property of AR models: Shocks to the system linger for many steps. Even though $Y_t$ only explicitly depends on $p$ previous values, those previous values each depended on their previous values. So the effect of a shock at time $t$ ripples forward for a long time. This is different from MA models where shocks die out after exactly $q$ steps.
Part B On the Exam: ACF/PACF & AR Models
06 Exam Q Type: "Is an AR(1) or AR(2) model more appropriate?"

Typical exam setup: You're shown a PACF plot and asked to determine the AR order.

The PACF plot shows a significant spike at lag 1, a significant spike at lag 2, and all subsequent lags are within the significance bands. Is an AR(1) or AR(2) model more appropriate?

Loses points

"AR(2) because the PACF has two spikes."

Right answer but too vague. Need to explain the reasoning.

Full marks

"An AR(2) model is more appropriate. The PACF shows significant spikes at lags 1 and 2, then cuts off (all subsequent lags fall within the significance bands). Since the PACF cuts off at lag $p$ for an AR($p$) model, the cutoff after lag 2 indicates AR(2)."
Decision rule:
1. Look at the PACF plot
2. Find the last lag with a significant spike
3. That lag number = the order $p$ of your AR model
4. Confirm: all lags after $p$ should be inside the blue bands
Your Turn

A PACF plot shows a significant spike at lag 1 only. Lags 2, 3, 4, ... are all within the significance bands. Meanwhile, the ACF plot shows a slow, gradual decay. What model is appropriate and why?

An AR(1) model is appropriate. The PACF cuts off after lag 1 (only lag 1 is significant, all others are within the bands), which is the signature pattern of an AR(1) process. The gradually decaying ACF confirms this — for AR($p$) models, the ACF tails off while the PACF cuts off at lag $p$.
07 Exam Q Type: ACF vs PACF pattern matching

You might be asked to distinguish between AR and MA models based on ACF/PACF plots. Here's the cheat sheet:

ModelACFPACF
AR($p$)Tails off (gradual decay)Cuts off after lag $p$
MA($q$)Cuts off after lag $q$Tails off (gradual decay)
Memory trick: AR and PACF both have the letter "R" — the PARtial autocorrelation is what cuts off for AR models. Or just remember: "AR → PACF cuts off" and the MA case is the opposite.
acf_pacf.R
# Always run both plots together
acf(data)   # check for gradual decay vs cutoff
pacf(data)  # check for cutoff at specific lag

# Fitting based on what you find:
library(forecast)
# AR(1): PACF cuts at 1
Arima(data, order = c(1, 0, 0))
# AR(2): PACF cuts at 2
Arima(data, order = c(2, 0, 0))
# MA(1): ACF cuts at 1
Arima(data, order = c(0, 0, 1))
Your Turn

You see an ACF that cuts off sharply after lag 3 (lags 4+ are within the bands) and a PACF that decays gradually. What type of model is this, and what order?

This is an MA(3) model. The ACF cutting off after lag 3 with a gradually decaying PACF is the signature pattern of an MA($q$) process with $q = 3$.
08 Exam Q Type: R code for fitting AR models

You may be asked to write the R code to fit a specific AR model. The function is Arima() from the forecast package.

arima_syntax.R
library(forecast)

# Arima(y, order = c(p, d, q))
#   y     = the time series data (vector or column)
#   order = c(p, d, q)
#     p = AR order
#     d = differencing (always 0 in this course)
#     q = MA order

# Example: Fit AR(2) to a vector called y
model <- Arima(y, order = c(2, 0, 0))

# To see coefficients and p-values:
library(lmtest)
coeftest(model)
Common exam mistakes:
• Writing arima() (lowercase) instead of Arima() (capital A)
• Getting the order wrong: c(p, d, q) — AR is first, MA is last
• Forgetting library(forecast)
• Confusing AR(2) = c(2, 0, 0) with MA(2) = c(0, 0, 2)

Part C Quiz: ACF/PACF & AR Models
09 4-Question Quiz
Q1

Time series data violates one of the LINE conditions for regression. Which condition is violated, and why?

Time series data violates the Independence condition (the "I" in LINE). In time series, each observation $Y_t$ depends on previous observations ($Y_{t-1}$, $Y_{t-2}$, etc.), so the observations are not independent of each other. This means the standard errors and p-values from ordinary linear regression (lm()) are unreliable.
Q2

You run acf(data) and see a slow, gradual decay across many lags. You run pacf(data) and see significant spikes at lags 1 and 2, with all subsequent lags inside the significance bands. What model is appropriate and what R code would you use to fit it?

An AR(2) model. The gradually decaying ACF combined with a PACF that cuts off after lag 2 is the signature pattern of an AR(2) process.

library(forecast)
model <- Arima(data, order = c(2, 0, 0))
Q3

For an AR(1) model $Y_t = c + \phi_1 Y_{t-1} + \eta_t$, explain why the PACF would show a significant spike only at lag 1, even though $Y_t$ is correlated with $Y_{t-2}$, $Y_{t-3}$, etc.

The PACF measures the direct correlation between $Y_t$ and $Y_{t-k}$ after removing the effect of intermediate lags. In AR(1), $Y_t$ is directly connected only to $Y_{t-1}$. Its correlation with $Y_{t-2}$ is entirely indirect — it passes through $Y_{t-1}$. Once you remove the effect of $Y_{t-1}$ (which is what the "partial" does), there is no remaining direct link between $Y_t$ and $Y_{t-2}$. So the PACF at lag 2 is zero.
Q4

What is the difference between $\eta_t$ and $Y_t$ in a time series model? Which one is independent across time and which is not?

$\eta_t$ is the shock (or innovation/noise) at time $t$ — it is independent across time, with each $\eta_t \sim N(0, \sigma^2)$. $Y_t$ is the observation at time $t$ — it is NOT independent across time because it is built from previous observations plus the current shock. In an AR(1): $Y_t = c + \phi_1 Y_{t-1} + \eta_t$, so $Y_t$ carries forward the influence of all past shocks through $Y_{t-1}$.

Part A Learn: Time Series Regression & Power (Lec 19)
01 Why lm() fails for time series data

When your data has time dependency, using lm() gives you wrong p-values. Here's why:

The core problem: lm() assumes the Independence condition in LINE. When residuals are autocorrelated (each residual is correlated with the previous one), the standard errors that lm() computes are too small. Smaller standard errors → bigger t-statistics → smaller p-values → you reject $H_0$ more often than you should.

This means lm() on time series data has inflated Type I Error. You think you're testing at $\alpha = 0.05$ but you're actually rejecting way more than 5% of the time when $H_0$ is true.

lm() on time series

  • Ignores autocorrelation in residuals
  • Standard errors are too small
  • P-values are too small (unreliable)
  • Type I Error is inflated
  • Rejects $H_0$ too often

Arima() on time series

  • Models the autocorrelation explicitly
  • Standard errors are correct
  • P-values are reliable
  • Type I Error is at the correct $\alpha$ level
  • Rejects $H_0$ at the right rate
From Lec 19: The lecture showed this with the NBA example — fitting lm(pts ~ opp_def, data = nba_df) failed to properly account for the MA(1) structure in the data, while Arima() with the correct order got it right.
02 Arima regression with xreg — adding predictors to time series models

Regular Arima(y, order = c(p, 0, q)) just models the time series structure. To do regression (test whether a predictor $X$ affects $Y$), add the xreg argument:

arima_regression.R
library(forecast)

# Time series regression with AR(1) and one predictor
model <- Arima(y, order = c(1, 0, 0), xreg = x)

# With multiple predictors: pass a matrix
X <- cbind(x1, x2, x3)
model <- Arima(y, order = c(1, 0, 0), xreg = X)

# NBA example from lecture (MA(1) with one predictor):
model2 <- Arima(
  y = nba_df$pts,
  order = c(0, 0, 1),
  xreg = nba_df$opp_def
)
ArgumentWhat it doesExample
yThe outcome (time series)nba_df$pts
orderc(p, d, q) — AR, differencing, MAc(1, 0, 0) for AR(1)
xregPredictor variable(s) — vector or matrixnba_df$opp_def
The model being fit:
$$Y_t = \beta_0 + \beta_1 x_t + \phi_1 Y_{t-1} + \cdots + \phi_p Y_{t-p} + \eta_t$$ This is a regression with a predictor $x_t$ plus AR terms to handle the time dependency. The xreg argument provides $x_t$, and the order argument specifies the AR/MA structure.
Watch out with multiple predictors: If you have categorical variables (like Home/Away), you need to convert them to numeric (0/1) before passing to xreg. The Arima function doesn't handle factors like lm() does.
03 coeftest() — getting p-values from Arima models

Unlike lm() where you use summary(model) to see coefficients and p-values, for Arima models you need coeftest() from the lmtest package.

coeftest_example.R
library(forecast)
library(lmtest)

# Fit an Arima regression
model <- Arima(y, order = c(2, 0, 0), xreg = time)

# Get the coefficient table with p-values
coeftest(model)

The output looks like this:

Estimate Std. Error z value Pr(>|z|) ar1 0.6423 0.0975 6.59 4.3e-11 *** ar2 0.2387 0.0989 2.41 0.016 * intercept 1.8914 0.8762 2.16 0.031 * time 0.0312 0.0089 3.51 0.00045 ***
Reading the output: The rows labeled ar1, ar2 are the AR coefficients ($\phi_1$, $\phi_2$). The row labeled with your predictor name (e.g., time) is the regression coefficient ($\beta_1$). The Pr(>|z|) column gives the p-value — this is the one you use to decide significance.

lm() workflow

model <- lm(y ~ x, data)
summary(model)

Arima() workflow

model <- Arima(y, order=c(p,0,q), xreg=x)
coeftest(model)

04 Power comparison: lm vs Arima — the simulation

Lec 19 ran a simulation to compare what happens when you use lm() vs Arima() on AR(2) data. The results are important and potentially tricky on the exam.

The key result:
• lm() has a higher rejection rate than Arima()
• But this is NOT a good thing — it's inflated Type I Error
• lm() rejects $H_0$ too often because its p-values are unreliable (too small)
• Arima() has the correct Type I Error rate
• So lm() only appears more "powerful" — it's actually just making more false rejections
Exam trap: A question might say "lm() rejected $H_0$ more often in the simulation — does this mean lm() is more powerful?" The answer is NO. Higher rejection rate when $H_0$ is true is inflated Type I Error, not power. Power is the rejection rate when $H_A$ is true, but only meaningful if the Type I Error rate is correct. Since lm() has wrong Type I Error, its "power" comparison is meaningless.

Misleading conclusion

"lm() is more powerful because it rejects more often"

Wrong — confuses inflated Type I Error with genuine power

Correct conclusion

"lm() rejects more often because it ignores autocorrelation, producing unreliable (too small) p-values. This is inflated Type I Error, not higher power. Arima() is the appropriate model."
05 Simulation code pattern — generating AR data and comparing models

The lecture showed a simulation pattern you should recognize. Here's the structure:

simulation_pattern.R
library(forecast)
library(lmtest)

# Function to generate AR(2) data with a covariate
gen_AR2_reg <- function(n, phi1, phi2) {
  x <- runif(n, 0, 10)
  eps <- rnorm(n + 2, 0, 3)
  y <- rep(0, n + 2)
  y[1] <- eps[1]
  y[2] <- phi1 * y[1] + eps[2]
  for (i in 3:(n + 2)) {
    y[i] <- 0 + 2 * x[i - 2] + phi1 * y[i - 1] +
            phi2 * y[i - 2] + eps[i]
  }
  y <- y[3:(n + 2)]  # discard burn-in
  data.frame(y = y, x = x, time = 1:n)
}

# Simulation: 1000 reps
reps <- 1000
lm_reject <- 0
arima_reject <- 0

for (i in 1:reps) {
  df <- gen_AR2_reg(100, 0.65, 0.25)

  # lm approach (ignoring time dependency)
  m1 <- summary(lm(y ~ x, data = df))
  if (m1$coefficients[2, 4] < 0.05) lm_reject <- lm_reject + 1

  # Arima approach (accounting for AR(2))
  m2 <- Arima(df$y, order = c(2, 0, 0), xreg = df$x)
  ct <- coeftest(m2)
  if (ct["df$x", 4] < 0.05) arima_reject <- arima_reject + 1
}

lm_reject / reps     # inflated rejection rate
arima_reject / reps   # correct rejection rate
Key elements to recognize:
• arima.sim(model = list(ar = phi), n = n) is a shortcut for generating AR data without a covariate
• The manual loop approach (shown above) is needed when you have a covariate $x$
• "Burn-in" values: the first 1-2 observations are discarded because the AR process needs previous values to get started
• Compare rejection rates at $\alpha = 0.05$ across many repetitions
arima_sim_shortcut.R
# Quick way to generate AR(1) data (no covariate)
y <- arima.sim(model = list(ar = 0.8), n = 100)

# AR(2) data
y <- arima.sim(model = list(ar = c(0.65, 0.25)), n = 100)

Full marks

Your Turn

Daily stock returns are modeled with lm(returns ~ market_index). A colleague points out that the ACF of the residuals shows significant spikes at lags 1 and 2. What's the problem and what should be done instead?

The significant ACF spikes at lags 1 and 2 indicate that the residuals are autocorrelated, violating the Independence assumption in LINE. The p-values from lm() are unreliable. Instead, use Arima(returns, order = c(2, 0, 0), xreg = market_index) to account for the AR(2) structure in the residuals, then use coeftest(model) for reliable p-values.
07 Exam Q Type: "Which model has higher power? Is this a good thing?"

Typical exam setup: A simulation compares rejection rates of lm() and Arima() on AR data.

A simulation generates 1000 datasets from an AR(2) process under $H_0$ (no true effect of $X$). lm() rejects $H_0$ 23% of the time, while Arima() rejects 5% of the time at $\alpha = 0.05$. Which model has higher power? Is this a good thing?

Loses points

"lm() has higher power because it rejects more often. This is good because we want to detect effects."

Completely wrong. Confuses Type I Error with power.

Full marks

"This simulation is under $H_0$, so the rejection rate is the Type I Error rate, not power. lm() rejects 23% of the time at $\alpha = 0.05$ — this is massively inflated Type I Error, meaning lm() makes false positive conclusions 23% of the time instead of the intended 5%. Arima() correctly rejects at ~5%, showing proper Type I Error control. The lm() result is not a good thing — it means almost 1 in 4 conclusions would be wrong."
Critical distinction:
• Power = rejection rate when $H_A$ is true (a good thing to maximize)
• Type I Error = rejection rate when $H_0$ is true (should be $\leq \alpha$)

If the simulation generates data under $H_0$ (no true effect), then a higher rejection rate is inflated Type I Error, NOT higher power. The lm() model only appears more powerful because its p-values are broken.
Your Turn

A student says: "I ran both lm() and Arima() on the same time series regression, and lm() gave me a smaller p-value. So lm() is the better model." What would you tell this student?

A smaller p-value from lm() does not make it the better model. Because lm() ignores the autocorrelation in the residuals, its standard errors are too small, which makes p-values artificially small. The Arima() model properly accounts for the time dependency, so its (larger) p-value is actually the reliable one. Using lm() here would inflate your Type I Error rate, meaning you'd make false positive conclusions more often than intended.

Typical exam setup: You're given a dataset and told the time series structure, then asked to write the appropriate R code.

Data is in player_df with columns pts (points scored) and opp_def (opponent defensive rating). You want to test whether opp_def is associated with pts, accounting for an AR(1) structure. Write the R code.

Answer:
answer.R
library(forecast)
library(lmtest)

model <- Arima(player_df$pts,
               order = c(1, 0, 0),
               xreg = player_df$opp_def)
coeftest(model)
Common mistakes:
• Forgetting xreg — without it you're just fitting an AR model with no predictor
• Using summary(model) instead of coeftest(model)
• Writing arima() (lowercase) instead of Arima() (capital A)
• Wrong order: AR(1) is c(1, 0, 0), not c(0, 0, 1) (that would be MA(1))

Q3

In a simulation under $H_0$ (no true effect), lm() rejects 18% of the time and Arima() rejects 4.8% of the time at $\alpha = 0.05$. A student concludes: "lm() has higher statistical power." Is this correct? Explain.

Incorrect. Since the simulation is under $H_0$ (no true effect exists), the rejection rate measures Type I Error, not power. Power is the rejection rate when $H_A$ is true. The lm() rejection rate of 18% at $\alpha = 0.05$ represents severely inflated Type I Error — it makes false positive conclusions 18% of the time instead of the intended 5%. The Arima() rate of 4.8% is close to the nominal $\alpha = 0.05$, indicating correct Type I Error control. The higher rejection rate from lm() is not genuine power — it's a consequence of unreliable p-values caused by ignoring autocorrelation.
Q4

What does coeftest(model) do, and why do we use it instead of summary(model) for Arima models?

coeftest(model) (from the lmtest package) produces a coefficient table showing the estimated coefficients, standard errors, z-values, and p-values for each parameter in the Arima model. We use it instead of summary(model) because summary() on an Arima object does not provide p-values for the coefficients. coeftest() gives us the significance tests we need to determine whether predictors (in xreg) are statistically significant.

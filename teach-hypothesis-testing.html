1. The Three Simulation Patterns

Every simulation in this course is ONE of three patterns

Same loop structure every time. Only the mean argument and what you count changes.

Type I Error Simulation

Q: How often does the test falsely reject $H_0$?

Key: Generate under $H_0$ TRUE

type1_sim.R
count <- 0
for (i in 1:10000) {
  samp <- rgamma(n = 5, shape = 1.2,
                 scale = 6 / 1.2)  # mean = 6 = null
  p <- t.test(samp, mu = 6)$p.value
  if (p < 0.05) count <- count + 1
}
TypeI <- count / 10000

Power Simulation

Q: How often does the test correctly reject $H_0$?

Key: Generate under $H_A$ TRUE

power_sim.R
count <- 0
for (i in 1:10000) {
  samp <- rnorm(n = 30, mean = 22,
                sd = 5)  # mean = 22 ≠ 20 (null)
  p <- t.test(samp, mu = 20)$p.value
  if (p < 0.05) count <- count + 1
}
power <- count / 10000

Bootstrap Type I Error Simulation

Q: How often does the bootstrap CI falsely exclude $\mu_0$?

boot_type1.R
count <- 0
for (i in 1:10000) {
  samp <- rnorm(n = 10, mean = 20, sd = 5)  # H0 true
  boots <- replicate(5000, mean(sample(samp, replace = TRUE)))
  ci <- quantile(boots, c(0.025, 0.975))
  if (20 < ci[1] || 20 > ci[2]) count <- count + 1  # 20 outside CI
}
type1_boot <- count / 10000
The ONLY difference between Type I Error sim and Power sim:
Type I Error: mean = null value ($H_0$ is true)
Power: mean = null value + delta ($H_A$ is true)
Everything else is identical.

2. Sign Test — Full Breakdown

What the Sign Test Does

Tests whether the median equals some value $m_0$. Compare to t-test which tests the mean.

Why this matters: for skewed data, mean $\neq$ median. The tests can give different answers.

Step-by-Step

  1. Subtract $m_0$ from every observation
  2. Count positives ($n_+$) and negatives ($n_-$). Ignore zeros (ties).
  3. Effective $n = n_+ + n_-$ (excluding ties)
  4. Under $H_0$: $n_+ \sim \text{Binomial}(n_{\text{eff}}, 0.5)$
  5. Two-sided p-value: $2 \times P(X \leq \min(n_+, n_-))$, capped at 1
sign_test_function.R
sign_test <- function(x, mu0) {
  signs <- x - mu0
  n_pos <- sum(signs > 0)
  n_neg <- sum(signs < 0)
  n_eff <- n_pos + n_neg           # exclude ties
  min_count <- min(n_pos, n_neg)
  p_val <- 2 * pbinom(min_count, size = n_eff, prob = 0.5)
  return(min(1, p_val))            # cap at 1
}

The Discreteness Problem

With $n = 7$, the achievable significance levels are fixed by the Binomial:

Rejection if $\min \leq k$$2 \times P(X \leq k)$
$k = 0$$2 \times 0.0078 = 0.016$
$k = 1$$2 \times 0.0625 = 0.125$
$k = 2$$2 \times 0.227 = 0.453$

Jumps from 0.016 to 0.125 — skips 0.05 entirely. The binomial is discrete, so you can only achieve certain $\alpha$ levels. With $n = 7$, the closest to 0.05 is 0.016.

Exam answer: "The Binomial distribution is discrete, so the achievable significance levels are a fixed set determined by $n$. With $n = 7$, no combination of outcomes gives a tail probability of exactly 0.05."

Sign Test vs t-Test: When They Disagree

From HW1: minutes played data ($n = 32$):

  • t-test: $p = 0.035$ → reject $H_0$
  • Sign test: $p = 0.377$ → fail to reject $H_0$

Why? t-test tests the mean, sign test tests the median. If data is skewed, mean $\neq$ median. Also, sign test throws away magnitude info (only counts signs), so it has less power than t-test when normality holds.

From HW1: Simulation showed t-test power ~0.75 vs sign test power ~0.55 with normal data. The t-test wins when conditions are met because it uses magnitude, not just direction.

3. Power — The Full Picture

Definition

$$\text{Power} = P(\text{reject } H_0 \mid H_0 \text{ is false}) = 1 - P(\text{Type II Error})$$

Target: 80%+ is "adequate." Power = 1.0 means you'd reject in every possible sample under $H_A$.

Four Factors (Memorize the Arrows)

$$n \uparrow \quad \delta \uparrow \quad \sigma \downarrow \quad \alpha \uparrow \qquad \Longrightarrow \qquad \text{Power} \uparrow$$
FactorDirection for more powerWhy
Sample size $n$$\uparrow$ increaseMore data = more evidence
Effect size $\delta$$\uparrow$ increaseBigger signal = easier to detect
Std dev $\sigma$$\downarrow$ decreaseLess noise = signal stands out
Significance $\alpha$$\uparrow$ increaseLower bar to reject = easier to reject
Common trap: "Decrease $\alpha$ from 0.05 to 0.01" sounds stricter and better, but it decreases power. Stricter threshold = harder to reject = more likely to miss real effects.

power.t.test() — When It Works and When It Doesn't

power_theory.R
# Full call with all arguments explicit
power.t.test(
  n           = 30,
  delta       = 2,          # true mean - null mean
  sd          = 5,
  sig.level   = 0.05,
  type        = "one.sample",
  alternative = "two.sided"
)
When it CANNOT be used: Data come from a non-normal distribution (Gamma, Uniform, etc.). power.t.test() assumes normality. For non-normal data, you MUST use simulation. This is asked on every quiz.

4. Key Wording & Named Concepts

Wording Cheat Sheet (Exam Loves These)

ConceptCorrect Wording
P-value"Probability of observing a test statistic as extreme or more extreme than the one observed, assuming $H_0$ is true"
Reject $H_0$"We reject $H_0$ / there is sufficient evidence to conclude..."
Fail to reject"We fail to reject $H_0$ / there is insufficient evidence." NEVER "We accept $H_0$"
Stat. vs practical significance"With large $n$, even tiny meaningless differences become statistically significant. Must also examine effect size."
Power = 1.0"The test would reject $H_0$ in every possible sample drawn under $H_A$"
Bootstrap Type I Error"Bootstrap CIs can have inflated or deflated Type I Error, especially for small samples. Non-parametric $\neq$ guaranteed 0.05."
Type I Error"Rejecting $H_0$ when $H_0$ is true" (false positive). Rate should = $\alpha$.
Type II Error"Failing to reject $H_0$ when $H_0$ is false" (false negative). Power $= 1 - \beta$.

Named Concepts the Exam Tests by Name

  • Statistical significance vs practical significance (Lec 5) — large $n$ makes everything "significant"
  • Parametric vs non-parametric — parametric (t-test) requires distributional assumptions; non-parametric (sign test, permutation, bootstrap) doesn't
  • Validity of a test — a test is "valid" when its Type I Error rate equals $\alpha$
  • Absence of evidence ≠ evidence of absence — failing to reject $H_0$ doesn't mean $H_0$ is true (could be low power)

Practice 5
Significance vs Effect Size

A poker player tracks 10,000 hands. Mean profit = $0.10/hand, $p = 0.22$. At 300 hands/hour, that's $30/hr.

(a) Should she reject $H_0$ (no profit) at $\alpha = 0.05$?
(b) Is this result practically meaningful?
(c) What does this illustrate — and how is it the opposite of the TSA example?
(a) No. $p = 0.22 > 0.05$, fail to reject.

(b) YES — $30/hr is meaningful income. The effect is real and large in practical terms.

(c) This is the flip side of statistical vs practical significance. TSA: statistically significant but practically meaningless (tiny effect, huge $n$). Poker: practically meaningful but NOT statistically significant (real effect, but high variance makes it hard to detect). Failing to reject $\neq$ $H_0$ is true. She may simply need more data (higher $n$) or lower variance.

Practice 7
One-sided P-value from Two-sided

Two-sided $p = 0.14$. Sample mean is below $\mu_0$. You want to test $H_A$: $\mu < \mu_0$.

(a) What is the one-sided p-value?
(b) At $\alpha = 0.10$, do you reject?
(a) $0.14 / 2 = 0.07$. Since the sample mean is in the direction of $H_A$, divide the two-sided p-value by 2.

(b) $0.07 < 0.10$ → reject $H_0$ at $\alpha = 0.10$.
Key: Only divide by 2 if the sample mean is in the direction of $H_A$. If it's in the opposite direction, the one-sided p-value is $1 - 0.14/2 = 0.93$.
Practice 8
Conceptual Rapid Fire

True or false:

(a) "Failing to reject $H_0$ means $H_0$ is true."
(b) "The sign test has more power than the t-test when data are normal."
(c) "A valid test has Type I Error rate equal to $\alpha$."
(d) "The bootstrap CI is guaranteed to have correct Type I Error because it's non-parametric."
(e) "$P(X \geq 5)$ = pbinom(5, n, p, lower.tail=FALSE)"
(a) FALSE — absence of evidence $\neq$ evidence of absence. Could be low power.
(b) FALSE — t-test has MORE power when normality holds (uses magnitude info).
(c) TRUE — definition of validity.
(d) FALSE — bootstrap can have inflated/deflated Type I Error, especially small $n$.
(e) FALSE — pbinom(5, ...) gives $P(X > 5)$, not $P(X \geq 5)$. Need pbinom(4, ..., lower.tail=FALSE).

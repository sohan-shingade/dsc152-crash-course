1. AR vs MA Error Structures

The Big Picture

In regular regression, errors are independent: each εt is a fresh random draw. In time series regression, errors can be correlated across time. There are two ways to model this:

AR — Autoregressive

Current error depends on previous errors

AR(1): ηt = φ1·ηt-1 + εt AR(2): ηt = φ1·ηt-1 + φ2·ηt-2 + εt

Key feature: Error feeds back into itself. Shocks echo and decay slowly.

Analogy: Momentum. A hot streak keeps going because yesterday's high carries forward.

MA — Moving Average

Current error depends on previous random shocks

MA(1): ηt = εt + θ1·εt-1 MA(2): ηt = εt + θ1·εt-1 + θ2·εt-2

Key feature: Shocks have a fixed lifespan. In MA(1), a shock affects only today and tomorrow, then vanishes.

Analogy: A hangover. Bad night affects today and maybe tomorrow, but by day 3 it's gone.

How to Tell Them Apart

Look at... AR(p) MA(q)
Formula clue Has ηt-1 (previous error) Has εt-1 (previous shock/innovation)
Shock duration Echoes forever (decaying) Dies after exactly q steps
ACF pattern Tails off (gradual decay) Cuts off after lag q
PACF pattern Cuts off after lag p Tails off
Memory trick: AR and MA are mirror images of each other in every way. AR: ACF tails off, PACF cuts off. MA: ACF cuts off, PACF tails off. They swap.

Visual: What AR(1) vs MA(1) Data Looks Like

Top: AR(1) with φ=0.8 — smooth, momentum-like. Bottom: MA(1) with θ=0.8 — choppier, shocks fade fast.

Check Yourself

A model has error term: ηt = εt - 0.55·εt-1

Q: Is this AR or MA? What order? How many time steps does a shock last?

MA(1). The error depends on εt and εt-1 (random shocks, not previous errors). A shock lasts exactly 1 additional time step, then vanishes. The -0.55 means 55% of the previous shock carries forward with reversed sign.

Check Yourself

A model has error term: ηt = 0.65·ηt-1 + 0.25·ηt-2 + εt

Q: Is this AR or MA? What order? What ACF/PACF pattern would you expect?

AR(2). The error depends on ηt-1 and ηt-2 (its own previous values). ACF would tail off (gradual decay). PACF would cut off after lag 2 (significant at lags 1-2, then drops to zero).

2. Arima() Syntax & the order Argument

The Arima() Function — Your New Best Friend

When errors have time structure, replace lm() with Arima():

Old Way (ignores time)

lm(pts ~ opp_def, data = df)

Assumes errors are independent. Wrong for time series.

New Way (models time structure)

Arima(y = df$pts, order = c(0, 0, 1), xreg = df$opp_def)

Models MA(1) errors. Correct standard errors.

The order = c(p, d, q) Cheat Sheet

order = c(p, d, q) ↑ ↑ ↑ AR order differencing MA order (always 0 in this class)
You want... order = Plain English
MA(1) errors c(0, 0, 1) 0 AR, 0 diff, 1 MA
AR(1) errors c(1, 0, 0) 1 AR, 0 diff, 0 MA
AR(2) errors c(2, 0, 0) 2 AR, 0 diff, 0 MA
AR(1) + MA(1) c(1, 0, 1) 1 AR, 0 diff, 1 MA
Key syntax differences from lm():
• No formula (y ~ x). Instead: y goes in y=, predictors go in xreg=
• xreg takes a vector (1 predictor) or matrix (multiple predictors)
• Need library(forecast) loaded first

Multiple Predictors? Use a Matrix for xreg

# Single predictor model <- Arima(df$pts, order = c(0,0,1), xreg = df$opp_def) # Multiple predictors — build a matrix X <- cbind(df$opp_def, df$MIN, df$TO) model <- Arima(df$pts, order = c(0,0,1), xreg = X)

Check Yourself

You have admission rate data (admit) over 50 years with a time predictor (year). ACF tails off, PACF cuts off after lag 2. Write the Arima call.

PACF cuts off after lag 2 → AR(2).
model <- Arima(admit, order = c(2, 0, 0), xreg = year)
AR(2) → first number is 2. Middle always 0. No MA → last number is 0.

3. coeftest() — Getting P-values

The Problem: Arima summary() Has No P-values

With lm(), you get p-values from summary(model). With Arima(), summary() does NOT give p-values. You need a separate function.

WRONG

model <- Arima(pts, order=c(0,0,1), xreg=opp_def) summary(model) # Shows coefficients and SEs # but NO p-values!

CORRECT

library(lmtest) model <- Arima(pts, order=c(0,0,1), xreg=opp_def) coeftest(model) # Shows coefficients, SEs, # z-values, AND p-values
Memorize this pattern — 3 lines every time:
1. library(forecast)
2. model <- Arima(y, order=c(p,0,q), xreg=X)
3. library(lmtest); coeftest(model)

Comparison: How You Get P-values in Each Model Type

Model Type Fit with P-values from Test stat
Linear regression lm() summary() t-value
Logistic regression glm() summary() z-value
Time series regression Arima() coeftest() z-value
Exam pattern: If they give you Arima output and ask "how did the researcher get these p-values?", the answer is coeftest(model) from the lmtest package. NOT summary().

4. Reading Arima Output Tables

Anatomy of an Arima + coeftest() Output

Here's a real output. Let's label every row.

EstimateStd. Errorz valuePr(>|z|)
intercept-30.1016.20-1.860.063
ma1-0.550.09-6.11<0.001
opp_def0.480.143.430.001

Row 1 — intercept: Same as lm(). The β0. Predicted y when all predictors = 0.

Row 2 — ma1: This is the time series parameter. It's θ1 in the MA(1) error model. This row does NOT exist in lm() output. It tells you the error structure strength.

  • ma1 → MA(1) parameter (θ1)
  • ar1 → AR(1) parameter (φ1)
  • ar1, ar2 → AR(2) parameters (φ1, φ2)

Row 3 — opp_def: Your actual predictor, same as lm(). This is β1. Interpret like regular regression: "For each 1-unit increase in opponent defensive rating, predicted points increase by 0.48, accounting for the MA(1) error structure."

The Key Interpretation Difference

Interpreting β1 (the predictor coefficient) works the same as in lm(). The only new thing is the time series parameter row.

Wording template for the exam:
"For each 1-unit increase in [X], [Y] is predicted to change by [b1] on average, after accounting for the [AR(p)/MA(q)] error structure."

You probably won't need to deeply interpret the ma1/ar1 coefficients themselves. Just know:

  • ma1 = -0.55 means 55% of the previous time step's random shock carries into the current error (with reversed sign)
  • If ma1 is significant (p < 0.05), the MA structure is real and you were right to use Arima() instead of lm()

What If There Are AR Terms Instead?

If you fit Arima(y, order=c(2,0,0), xreg=time), the output looks like:

EstimateStd. Errorz valuePr(>|z|)
ar10.650.106.50<0.001
ar20.250.102.500.012
intercept2.100.802.630.009
time0.030.013.000.003

Two highlighted rows for φ1 and φ2. The time row is your actual predictor — that's what you interpret for inference.

Summary: Everything on One Cheat Sheet

Time Series Regression — Complete Reference

THE MODEL: y_t = β_0 + β_1·x_t + η_t where η_t follows AR(p) or MA(q) instead of being independent ERROR STRUCTURES: AR(p): η_t = φ_1·η_{t-1} + ... + φ_p·η_{t-p} + ε_t [shocks echo] MA(q): η_t = ε_t + θ_1·ε_{t-1} + ... + θ_q·ε_{t-q} [shocks die after q steps] IDENTIFICATION (from Lec 17): AR(p): ACF tails off, PACF cuts off after lag p MA(q): ACF cuts off after lag q, PACF tails off R CODE — 3 LINES: library(forecast) model <- Arima(y, order = c(p, 0, q), xreg = X) library(lmtest); coeftest(model) order = c(p, d, q): p=AR, d=diff(always 0), q=MA KEY TAKEAWAY: Ignoring time dependency → incorrect inference → loss of power Arima() with correct structure → proper SEs → trustworthy p-values
